# 쿠팡 제품기획

경쟁사 불만을 그대로 카피하지 말고 `반복 불만 → 구매 기준 → 우리 SKU가 해결 가능한가 → 근거 → 제한된 주장`으로 변환한다.

## 작업 순서

1. 공급처 상세 URL 하나만으로 조사를 시작한다. 기존 소싱 후보·가격안이 있으면 잠정 근거로 보존하되 조사 전 최종 오퍼로 확정하지 않는다.
2. Browser Harness로 공급처의 상세 본문·옵션·상품정보·공개 후기·문의를 조사해 색상·사이즈·구성·소재·관리법, 단가·MOQ·구매 단위·배송 조건과 제품 식별 정보를 찾는다. 상세 이미지 안의 표·치수는 육안으로 판독하고 URL·조사시각·근거 위치를 남긴다.
3. 공급처 사실에서 검색어를 만들고 쿠팡의 관련 경쟁상품 5개 이상과 접근 가능한 별점 1~3점 리뷰를 직접 조사한다. 경쟁상품 URL·리뷰 근거·조사시각·표본 범위를 보존하고 문제 유형·빈도·반품 위험으로 묶는다.
4. 조사 결과를 사용자에게 먼저 보고 `확인 사실 / 공급처 주장 / 충돌 / 미확인 / 접근 차단`을 분리한다. 공급처 옵션은 실제 수령 SKU로 간주하지 말고 실물 사진·라벨·사용자 선택과 대조한다.
5. 각 경쟁 문제를 실제 SKU가 해결할 수 있는지 `가능 / 조건부 / 불가 / 미확인`으로 판정하고 검사서·시험성적서·간이 실험·실측·라벨 근거를 연결한다.
6. **오퍼 결정 게이트**에서 사용자와 타깃, 실제 판매 SKU, 핵심 문제·소구, 판매가·묶음·판매 옵션, 포지셔닝, 브랜드 방향과 1차 실험 범위를 함께 결정한다. 이 단계는 사용자와 정하는 오퍼 결정 게이트이며, 조사 전에 이 값을 필수 입력으로 요구하거나 임의 확정하지 않는다.
7. 결정된 타깃, 문제, 포지셔닝, 오퍼, 옵션, 사이즈, 관리법, 게시 가능 주장과 금지 주장을 `product-plan.json`에 기록한다.
8. 사용자에게 계획 전체를 한 번에 보여주고 현재 파일 해시에 대한 명시적 승인을 받는다.

## 입력 경계

- 공급처 상세 URL 하나만으로 조사를 시작하고, 색상·사이즈·구성·소재·관리법을 사용자가 먼저 전사하도록 요구하지 않는다. 공급처 공개 근거를 끝까지 확인한 뒤 실제 SKU 확정에 필요한 사진·라벨·측정만 구체적으로 요청한다.
- 사용자에게 리뷰 URL이나 조사 파일 경로를 입력받지 않는다. 경쟁사 저평점 리뷰 수집과 근거 정리는 Codex의 조사 작업이다.
- 사용자에게 이미지 폴더 경로를 입력받지 않는다. 대화에 첨부된 사진이나 `folderMap.sourceAssets`의 기존 파일을 읽고, 부족하면 필요한 촬영 구도·라벨 면을 구체적으로 요청한다.
- 공급처 페이지에서 찾은 값도 실수령 SKU와 일치하는지 구분한다. 값마다 공급처 원문·실측·라벨·사용자 확인 중 하나 이상의 Evidence ID를 연결하지 못하면 확정 사실로 승격하지 않는다.
- 검색 작업마다 사용자 브라우저와 분리된 임시 세션을 사용하고, 완료·차단·예외와 관계없이 작업이 연 탭과 소유 브라우저 프로세스를 닫은 뒤 남은 작업 탭이 없는지 확인한다. 표시형 브라우저로 자동 전환하지 않는다.
- 로그인·CAPTCHA·접근 통제를 우회하지 않는다. 공개 근거를 확보하지 못하면 추정하지 않고 차단 상태와 재개 조건을 먼저 보고한다.

## 1차와 2차

- `phase_1`: 최소 SKU 사실과 가능한 간이 실험만 수행한다. 실험 불가 소구는 미검증 가설로 남기고 판매 주장에는 쓰지 않는다.
- `phase_2`: 사용자가 제공한 캠페인별 14일 내 판매 2개 이상과 ROAS 400% 이상을 모두 충족한 경우에만 진입한다. 자사 후기, 다수 샘플, 검사서 적용 범위, 심화 실험, 실제 사이즈·관리법을 다시 검증한다.
- 필수 안전·법정 표시·제품 동일성은 2차로 미루지 않는다.

## Acceptance Criteria

- 핵심 소구마다 경쟁 불만·SKU 해결 가능성·Evidence ID 연결: 100%
- `불가/미확인` 소구의 판매 카피 사용: 0건
- 수치·성능 주장마다 적용 SKU가 확인된 근거: 100%
- 사이즈 값마다 단위·측정 시작점·끝점·허용 오차 상태 기록: 100%
- 관리법마다 케어라벨·공급처·실험 중 하나의 출처 연결: 100%
- 사용자 승인이 현재 `product-plan.json` SHA-256에 결속됨: 필수

상세 산출물과 롤백은 [references/product-planning-contract.md](references/product-planning-contract.md)를 따른다.
