## 3. 10장 구매 서사와 장별 자산 전략

모든 제목·설명·배지·치수·색상명·사양·출처는 네이티브 HTML로 둔다. 실제 제품 사진, 시험지, GIF·영상만 외부 자산으로 관리한다.

| 장 | 구매 질문·역할 | HTML 헤드라인 | 현재 자산의 역할 | 최종 제작 조건 |
|---:|---|---|---|---|
| 01 | 무엇이 다른가 · `hero` | **햇빛은 가리고, 숨길은 열고** | 인물 포함 5개는 구도 참고만 | 사람 없는 실제 SKU 제품 단독 정면·사선 P0 촬영 |
| 02 | 가리면 답답하지 않을까 · `problem_criterion` | **가림과 호흡, 둘 다 놓치지 않도록** | 네이티브 HTML 기준 카드, 경쟁사 픽셀 미사용 | 실제 제품 단독과 비제품 추상 도식만 사용 |
| 03 | 무엇으로 확인할까 · `solution_overview` | **시험·구조·실측, 세 가지 답** | 시험지·상품정보표를 증거 유형으로 연결 | 제품 전체·타공·실측을 각각 실제 근거와 연결 |
| 04 | UPF50+ 근거는 · `uv_proof` | **UPF50+, 시험 결과를 그대로** | Intertek 원본 직접 사용 | 보고서 번호·3/7쪽·UPF50+ 식별, 임의 차단율 금지 |
| 05 | 호흡부 구조는 · `air_detail` | **입·코 주변 호흡부 타공** | 색상별 착용 사진은 타공 위치 참고만 | 사람 없는 실제 SKU 타공 매크로 P0 촬영; 효용 카피는 호흡 QA 통과 후 |
| 06 | 귀·얼굴선·후면은 · `fit_detail` | **원단형 귀 구멍과 곡선형 얼굴선** | 측면·후면 인물 사진은 촬영 각도 참고만 | 제품 단독과 마네킹 정·측·후면 P1 촬영; 압박 없음 단정 금지 |
| 07 | 목까지 이어지는가 · `cover_proof` | **총 길이 25cm, 목까지 이어지는 롱커버** | 정면·후면 사진은 길이 구도 참고만 | A·D 기준점이 보이는 실제 자 실측 P0 촬영 |
| 08 | 색상 선택은 · `option_friction` | **블랙부터 핑크까지, 네 가지 선택** | 상품정보표는 색상명 원천, 착용 사진은 색 참고만 | 4색 제품 단독 동일 구도·화이트밸런스 P0 촬영 |
| 09 | 소재·치수·관리는 · `purchase_information` | **구매 전 확인할 제품 정보** | 상품정보표의 76/24·중국·제조사·판매원을 HTML로 전환 | 포장·케어라벨·A~E 실측 확인; 관리법은 라벨 전까지 비노출 |
| 10 | 핵심을 다시 보면 · `recap` | **UPF50+·호흡부 타공·25cm 롱커버** | 04·05·07의 서로 다른 증거를 회수 | 01과 다른 제품 단독 후면·사선, 승인 주장만 요약 |

기능 증거가 주인공인 장은 03~09의 7장이다. 01과 10도 제품이 중심이며 실제 인물 중심 라이프스타일 페이지는 만들지 않는다.

## 4. 이번 자산 검토로 바뀐 표현

- 06장의 `4.5cm 귀 리본`은 현재 사진에서 별도 리본이 아니라 원단형 귀 구멍으로 보이므로, 실측 전에는 **원단형 귀 구멍과 곡선형 얼굴선**으로 표현한다.
- E 4.5cm는 사용자 확인 수치로 원장에 유지하되 실제 SKU의 측정 기준점과 명칭을 확인한 뒤 09장 HTML 사양표에 넣는다.
- `구멍이 있어 숨쉬기 편하게`는 1차 호흡·습기 간이 QA를 통과한 경우에만 05장 보조 카피로 승격한다. 통과 전에는 `입·코 주변 호흡부 타공 구조`까지만 사용한다.
- 관리법은 현재 `UNKNOWN`이므로 09장 자리만 확보하고 세탁·건조법을 임의 작성하지 않는다.
- 미승인 브랜드명 `에어베일`과 로고는 이미지·HTML 어디에도 노출하지 않는다. 승인 제품명 `숨트임`만 사용할 수 있다.

## 5. 이미지 + HTML 조립 원칙

| 레이어 | 내용 | 수정·QA 방식 |
|---|---|---|
| 이미지 | 실제 제품 단독, 타공·봉제·실측 매크로, 4색, Intertek 원본 | 자산별 해시·동일성·제품 왜곡 QA 후 실패 자산만 교체 |
| HTML | 모든 한글 카피, 증거 칩, 수치, 색상명, 사양표, 출처·한정 문구 | 문구·값만 수정하고 해당 장만 재렌더링 |
| 모션 | 타공 회전 GIF, 마네킹 롱커버 GIF, 8~12초 제품 증거 영상 | 첫 프레임 정적 대체본과 함께 자산 단위 QA |
| 정적판 | 승인된 HTML을 800×2400px 10장으로 렌더 | 각 장 OCR·정렬·제품 동일성·광고표현 QA 후 통합 QA |

## 6. QA 게이트

### 소재별 QA

- 파일 ID·해시·출처·주장 ID 연결
- 실제 인물 픽셀, 생성 제품, 임의 로고·글자·구멍·봉제 변화 0건
- 실제 SKU 실루엣·타공·귀 구멍·후면·롱커버·4색 동일성
- 시험지 원본 무변형과 한정 문구 연결
- 효용 카피의 호흡 QA 상태, 치수의 실제 측정 상태, 혼용률의 라벨 상태 확인

### 조립 후 통합 QA

- `문제/기준 → 3증거 → 시험 → 타공 → 착용 구조 → 롱커버 → 색상 → 정보 → 요약` 순서
- 이미지와 HTML 주장 간 모순·중복·누락 0건
- 모바일 오버플로·잘림·깨진 링크·대체 텍스트 누락 0건
- 미승인 브랜드, 냉감·KC·남녀공용·의료·정량 호흡 표현 0건
- 800×2400px 10장 파일명·순서·OCR·정렬·제품 왜곡·광고표현 통과
