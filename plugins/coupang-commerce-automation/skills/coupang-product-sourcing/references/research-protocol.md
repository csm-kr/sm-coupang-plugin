# 조사 프로토콜

Browser Use로 연 상품·검색 탭은 필요한 값과 근거 URL을 기록한 즉시 닫는다. 접근 차단, 로그인 벽 또는 CAPTCHA가 나타나도 상태를 기록한 뒤 탭을 닫고 우회하지 않는다.

## 후보 풀

- 기본 URL: `https://domeggook.com/main/item/itemPopular.php`
- 전체와 6개 대분류의 TOP 150을 후보 풀로 사용한다.
- `도전! 베스트`와 카테고리 TOP 150을 `pool_type`으로 구분하고 기본 표본은 각각 20%, 80%를 목표로 한다.
- 6개 대분류는 가능한 한 균등하게 포함한다. 비어 있는 카테고리 몫은 다른 카테고리에 재배분한다.
- `국산/국내산`, `국외산`, 원산지 미확인을 별도 층으로 수집한다. 기본 탐색 목표는 국내 45%, 국외 45%, 미확인 10%이며 실제 풀에 해당 층이 없으면 존재하는 층으로 재배분한다.
- 원산지 미확인은 탐색에는 포함할 수 있지만 상세 검증 전에는 SHORTLIST로 승격하지 않는다.
- 각 목록에서 순위, 순위 변동, 상품명, 표시 공급가, MOQ, 상세 URL을 수집한다.
- 상세페이지에서 구간별 단가, 실제 배송 조건·금액, 재고, 출고일, 옵션, 원산지, 모델명, 이미지 사용 허용, 공급사 상태를 보완한다.
- `주문 시 결제`처럼 금액이 노출되지 않는 비용은 0으로 처리하지 않고 `UNKNOWN`으로 둔다.
- TOP 150은 도매꾹 내부 인기 신호일 뿐 최종 소비자 수요가 아니다. 쿠팡 검색·판매·리뷰 신호가 없으면 통과시키지 않는다.

## 도매 후보 수집

규제 위험이 낮은 생활용품을 기본 범위로 삼고 후보 30개를 목표로 한다. 공급가 2,000~20,000원, MOQ 1~5개, 옵션 1~3개, 목표 소비자가 9,900~39,900원은 탐색 우선순위일 뿐 하드 필터가 아니다.

상품 페이지에서 공급가, MOQ, 배송비, 옵션, 공급사, 인증 표기, 원본 이미지 수와 사용 조건, 품절·가격변경·배송지연 공지 방식을 확인한다. 목록 카드의 값은 상세페이지에서 다시 확인한다.

## 하드 필터

다음은 점수와 관계없이 탈락 사유다.

- 공급가, MOQ, 도매 배송비 중 하나라도 미확인
- 초기 매입액 `공급가 × MOQ`가 설정 한도 초과
- MOQ가 설정 한도 초과
- 옵션 수가 설정 한도 초과 또는 옵션 구조 미확인
- 상표권, 캐릭터, 브랜드 도용 가능성
- 필요한 인증·시험·허가 여부 미확인 또는 부적합
- 파손, 누수, 사이즈 반품 같은 높은 위험을 해소할 근거 없음
- 상품 사진과 실제 공급상품의 일치 여부 불명확
- 사용 가능한 상품 이미지 부족 또는 사용권 제한
- 공급사가 품절, 가격변경, 배송지연 정보를 제공하지 않음

자동 감지로 확정하기 어려운 사유는 `manual_hard_failures`에 코드, 이유, 근거 URL을 기록한다. 규제 중요 카테고리는 탈락으로 꾸미지 말고 `separate_review`로 분리해 SHORTLIST를 막는다.

## 쿠팡 검색

후보마다 검색어 세 종류를 만든다.

1. `category`: 소비자가 실제로 찾는 대표 카테고리
2. `appeal`: 형태, 사용장면, 핵심 장점을 포함한 세부 검색어
3. `identical`: 모델명, 도매 상품명의 고유 표현, 이미지에서 확인한 고유 문구

각 검색어의 랭킹순 첫 화면에서 최대 20개를 수집한다. 광고는 `is_ad: true`로, 일반 결과는 `false`로 표시한다. 판매량순을 공개 UI에서 선택할 수 있을 때만 별도 시도하고, 불가능하면 접근 불가로 기록한다.

각 결과에서 상품명·URL·노출 위치·가격·구성수량·개당가격·평점·리뷰수·최근 구매 수·로켓·무료배송·판매자·브랜드·반복 소구·동일성·이미지 재사용을 기록한다. 가격·리뷰·최근 구매 신호는 같은 시각의 관찰값으로 묶는다. 판매 근거가 없는 등록가는 가격 중앙값에서 제외하되 제외 수를 보존한다.

로켓 배송 신호와 판매 주체는 분리한다. `판매자로켓`은 외부 로켓그로스 판매자의 경쟁상품이므로 진입 가능하다. 상품 상세에서 판매자가 쿠팡으로 확인되는 동일·근접상품만 탈락시킨다. 배지만 있고 판매자를 확인하지 못하면 검증 미완료로 둔다.

## 수요 신호

다음 신호를 함께 본다.

- 공개된 최근 구매자 수 또는 구매 배지
- 최근 30일·90일 리뷰 증가와 작성 간격
- 랭킹순·판매량순 노출 위치
- 대표·세부 검색어에서 반복 상위 노출
- 여러 판매자에게 분산된 리뷰와 최근 구매 흔적
- 신규 상품의 최근 리뷰 축적
- 할인 없이 유지되는 노출과 리뷰 증가
- 계절성의 현재 위치와 전년 동기 추세

리뷰 총량만으로 현재 수요를 단정하지 않는다. 최근 리뷰를 보려면 상품 페이지의 리뷰 정렬·날짜를 직접 확인하고, 표본 수와 확인 기간을 근거 메모에 적는다.

## 동일상품 묶기

동일성 판정을 가격 수집보다 먼저 수행한다. 공급처와 쿠팡의 대표·상세 이미지, 외형 구조, 규격, 모델, 옵션, 고유 문구가 모두 부합하는지 확인한다. 같은 검색어에 노출되더라도 구조·브랜드·규격이 다르면 `similar`로 두며, 그 가격을 완전 동일 상품의 직접 가격 제약이나 단독 탈락 근거로 사용하지 않는다.

판매자별 원자료 행은 보존한다. 다음 중 둘 이상이 일치할 때 동일·거의 동일 상품군으로 묶는다.

- 모델명 또는 고유 규격
- 외형과 구성
- 대표 이미지 또는 상세 이미지
- 도매 상품명의 고유 문구
- 상세페이지 카피와 이미지 순서

색상만 다른 동일 모델은 같은 상품군으로 본다. 기능과 구조가 다른 대체재는 `similar`로 둔다.

## 조사 완결성

- 세 검색어의 시도 상태와 결과 URL이 모두 있어야 한다.
- `category`와 `appeal` 검색은 차단되지 않고 실제 결과를 확인해야 한다.
- 수집한 모든 결과에 광고 여부와 조사시각이 있어야 한다.
- 가격 근거는 실제 쿠팡 상품 또는 검색결과 URL이어야 한다.
- CAPTCHA나 차단을 만나면 우회하지 않고 `blocked`로 남긴다.
- 조사 완결성이 부족한 후보는 총점이 높아도 `WATCH`를 넘을 수 없다.
