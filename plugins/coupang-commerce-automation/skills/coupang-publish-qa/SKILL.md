# 쿠팡 게시 전 QA

파일 존재만으로 통과시키지 않는다. 현재 계획·자산·HTML 해시에 묶인 자동 QA와 육안 QA가 모두 통과해야 한다.

## 검수 순서

1. 제품기획·콘텐츠기획·스토리보드 승인 해시가 현재 파일과 일치하는지 확인한다.
2. 각 소재의 파일·계보·제품 동일성·주장-근거·시각 품질을 검수한다.
3. 실제 브라우저 360px·800px에서 모듈 순서, claim/asset ID, 피사체·핵심 부위 가시율을 수집한다.
4. 한글 줄바꿈·고아행·오버플로·겹침을 별도 좌표 QA로 검사한다.
5. 접근성, 의미 있는 DOM 순서, 광고 표현, 채널 규격, 정적 대체 자산을 검수한다.
6. 실패 항목만 가장 가까운 스토리보드·자산·HTML 단계로 되돌린다.

## Acceptance Criteria

- 모듈 DOM 순서와 승인 스토리보드 순서 일치: 100%
- claim ID와 asset ID 연결 일치: 100%
- 피사체 가시율: 95% 이상
- 핵심 부위 가시율: 100%
- 주장-이미지 연관성: 모듈별 80/100 이상
- 소재 자동·육안 QA: 전 항목 pass
- 360px·800px 타이포 오류·strict 경고: 0건
- 끊어진 자산·누락 대체 텍스트·JS 예외: 0건
- 게시 불가 주장·미승인 브랜드·실제 사람 픽셀: 0건

기계 결과와 육안 결과 중 하나라도 실패하면 `completed`로 승격하지 않는다. 기존 검증 명령은 `$coupang-detail-page-generator`의 스크립트를 사용한다. 상세 판정표는 [references/acceptance-matrix.md](references/acceptance-matrix.md)를 따른다.
