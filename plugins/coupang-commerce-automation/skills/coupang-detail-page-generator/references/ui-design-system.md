# 쿠팡 상세페이지 UI 디자인 시스템

## 원칙

UI 가이드는 이미지 생성 전에 만든다. 브랜드 시스템의 상수를 상세페이지에 번역하고, 10장의 반복 요소와 페이지 역할별 변주를 함께 잠근다. 페이지별 프롬프트가 서로 다른 미니 브랜드를 만들지 않게 한다.

## 1. 캔버스

- 워크플로 5.3 HTML 정본: 최소 360px, 최대 800px 반응형, 모듈 높이는 콘텐츠에 따라 가변
- 워크플로 5.2 정적 이미지: `800×2400px` 10장 고정
- 워크플로 5.0 이하 기존 프로젝트는 기존 매니페스트 규격을 보존한다.
- 좌우 안전 여백: 64px 이상
- 주요 카피 상하 안전 여백: 96px 이상
- 첫 콘텐츠 시작: y 140px 이내
- 의도된 콘텐츠 세로 점유율: 70% 이상
- 사진·제품·정보 그래픽 점유율: 55~85%
- 간격 체계: 8px 기반

## 2. 브랜드 상속

`brand-system.json`에서 다음을 가져온다.

- 핵심 배경·텍스트·강조 컬러의 역할
- 한글 패밀리와 굵기
- 사진의 빛·거리·접지·매크로 원칙
- 카드·번호·선·모서리·그림자
- 브랜드 기억 장치
- 목소리와 금지 표현

카테고리 강조색과 사용 상황은 제품별 변수다. 브랜드 코어 컬러, 한글 패밀리, 기억 장치, 증거 태도는 상수다.

## 3. 타이포

| 역할 | 최종 800px 목표 | 굵기 | 줄 수 | 정렬 |
|---|---:|---|---|---|
| 히어로 | 74~94px | Black | 1~2줄 우선, 최대 3줄 | 중앙 또는 좌측 |
| 섹션 제목 | 60~78px | Black | 1~2줄 우선, 최대 3줄 | 역할 축 |
| 본문 | 26~34px | Regular/Medium | 최대 2문장 | 제목 축 |
| 라벨·칩 | 19~25px | Bold | 1줄 | 카드 축 |
| 단계 숫자 | 42~66px | Black | 1줄 | 단계 축 |

한글은 한 패밀리, 굵기는 Regular·Bold·Black 세 단계 이내다. 제목의 `emphasis` 구절 하나만 강조색으로 바꾼다. 세로로 늘어난 글리프, 짧은 둘째 줄 과대 확대, 3줄을 넘는 제목, 약한 명도 대비를 금지한다.

## 4. 정렬 축

- 중앙정렬 블록은 배지·제목·서브카피의 광학 중심을 하나의 세로축에 둔다. 최종 800px에서 16px 이내 오차를 목표로 한다.
- 비교·단계·정보 페이지는 좌측축 또는 카드축을 사용한다. 10장을 전부 중앙정렬하지 않는다.
- 반복 카드는 같은 폭·높이·거터·모서리·내부 여백·제목 시작선을 공유한다.
- 정보표는 `라벨 열 / 구분점 열 / 값 열`을 고정한다. 줄바꿈된 값도 같은 값 열에 행걸이 정렬하며 시작축 편차 8px 이내다.

## 5. 컴포넌트 계약

| 컴포넌트 | 필수 규칙 | 금지 |
|---|---|---|
| 비교 카드 | 같은 크기·거터, 비교 기준 1개 | 근거 없는 우열·경쟁 비방 |
| 원형 확대 | 원본 제품 위치와 연결, 2~3개 이내 | 같은 사진 확대 반복 |
| 단계 카드 | 숫자·사진·캡션 시작선 반복 | 제품 구조가 단계마다 바뀜 |
| 기능 증거 | 접촉·행동·반응이 실제 제품에 보임 | 아이콘·물방울 그래픽만 사용 |
| 주의 카드 | 충분한 대비와 짧은 문장 | 공포 조장 |
| 사이즈표 | 고정 열과 단위, 출처 있는 값 | 추정 치수 |
| 요약 카드 | 서로 다른 핵심 3개 | 같은 기능 재포장 |

## 6. 사진과 그래픽

- 제품이 주인공인 페이지는 제품을 작게 띄우지 않는다.
- 제품 그림자는 광원과 접지면에 맞춘다.
- 기능 접촉은 제품 표면·결합부·변형 자체에서 보이게 한다.
- 물방울·빛·화살표는 보조이며 실제 증거를 대신하지 않는다.
- 얼굴·손·제품·라벨·인셋 위에 텍스트를 올리지 않는다.
- 브랜드 기억 장치는 10장 전체에서 같은 형태와 의미로 반복한다.

## 7. 페이지 반복과 변주

반복: 팔레트 역할, 한글 패밀리, 카드 재질, 번호, 선, 그림자, 증거 표현 태도.

변주: 풀 프레임, 플랫레이, 접촉 매크로, 원형 확대, 연속 동작, 보관 비교, 단계 스트립, 모자이크, 정보표, 제품 중심 클로징.

같은 모듈을 연속 사용하거나 같은 모델·제품 각도를 크롭만 바꿔 반복하지 않는다.

## 8. UI 승인

제작 전 다음을 모두 확인한다.

- `ui-guide.md`가 `brand-system.json`을 참조한다.
- 브랜드 상수와 제품 변수가 분리되어 있다.
- 모든 UI 토큰에 값·용도·브랜드 근거가 있다.
- 10장 각각의 정렬 축과 컴포넌트가 페이지 계획에 지정되어 있다.
- 스타일 프레임 또는 제품 보존 시험 페이지에서 폰트·카드·사진 비율을 확인했다.

UI 가이드 없이 이미지부터 생성하거나, 템플릿의 빈 토큰을 남긴 채 제작으로 넘어가면 실패다.
