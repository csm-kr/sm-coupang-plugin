# 상세페이지와 함께 만드는 브랜드 시스템

## 브랜드의 역할

브랜드는 로고·이름보다 먼저 `누구의 어떤 불안을 어떤 증거 태도로 줄이는가`를 반복하는 운영 규칙이다. 경쟁사 스타일을 섞어 평균적인 쇼핑몰 화면을 만들지 않는다. 모든 제품 프로젝트에는 브랜드명 제안이 있어야 한다.

## 브랜드 상태

- `established`: 사용자가 이름·로고·가이드 또는 명시적 기준을 제공함. 핵심 자산을 변경하지 않는다.
- `working_draft`: 브랜드가 없거나 불완전함. 제품·고객·시장 근거로 작업용 방향과 제품별 제안명을 만들되 이름·로고를 최종 이미지에 넣지 않는다.
- `approved`: 사용자가 이름, 핵심 약속, 목소리, 시각 시스템의 사용을 승인함.

`working_draft`도 팔레트·사진·컴포넌트·말투에는 사용할 수 있다. 제안명은 보고서에서 명시하지만, 승인되지 않은 이름·슬로건·로고는 이미지에 사용할 수 없다.

## 표현 우선순위

1. `practical_evidence` — 실용 증거형: 고객이 실제 효용을 즉시 이해하도록 접촉·행동·구조·변형·사용 전후를 먼저 보인다.
2. `professional_function` — 전문 기능형: 작동 원리와 구조를 정확한 용어·매크로·도식으로 보강한다.
3. `emotional_lifestyle` — 감성 생활형: 앞선 증거를 해치지 않는 범위에서 사용 장면·분위기·소유 만족을 더한다.

세 우선순위를 섞어 평균내지 않는다. 장별 역할은 달라도 구매 이유는 1번에서 시작하고, 2번으로 신뢰를 높인 뒤, 3번으로 생활 맥락을 완성한다.

## 제품별 브랜드명 게이트

1. 제품 사실·대표 고객 불안·핵심 효용에서 이름의 의미 영역을 정한다.
2. 후보를 3개 이상 만들고 뜻·발음·기억성·카테고리 확장성·오인 위험을 비교한다.
3. 최종 제안 1개를 선택하고 `brand-name-candidates.md`와 `brand-system.json`에 같은 이름을 기록한다.
4. 이름 상태는 `proposed`, `approved`, `established` 중 하나다.
5. `proposed`는 최종 보고에서 말하되 `name_usage_allowed: false`로 상세 이미지 노출을 막는다.
6. `approved` 또는 `established`만 상세 이미지에 이름을 사용할 수 있다. 로고 사용은 별도의 로고 정책을 따른다.

상표 등록 가능성이나 중복 여부를 확인하지 않았다면 이름의 법적 사용 가능성을 단정하지 않는다. `위험·중복 확인` 칸에 확인 범위와 미확인 상태를 남긴다.

## 브랜드 정반합

1. `category_codes`: 고객이 카테고리를 즉시 이해하는 익숙한 색·장면·정보 구조
2. `market_cliches`: 경쟁사가 반복해 차별성이 사라진 표현과 장식
3. `own_truth`: 우리 제품과 운영에서 반복 증명할 수 있는 사실
4. `customer_tension`: 고객이 구매 전에 해결받고 싶은 가장 큰 불안
5. `distinctive_synthesis`: 익숙한 정보 구조 안에서 우리만 반복할 약속·목소리·시각 장치

예쁜 컬러를 먼저 고르지 않는다. `고객 불안 → 브랜드 약속 → 이유가 되는 제품 증거 → 표현 방식` 순서로 정한다.

## brand-brief.md

- 브랜드 상태와 기존 자산
- 핵심 타겟과 사용 상황
- 대표 고객 불안
- 브랜드가 맡을 문제 영역
- 한 문장 포지셔닝
- 핵심 약속 1개
- 이유가 되는 사실 3개
- 성격 3개와 배제할 성격 3개
- 카테고리 관습, 시장 클리셰, 차별화 기회
- 제품별 브랜드명 후보 3개 이상과 최종 제안 1개
- 이름 상태와 이미지 사용 승인 여부
- 사용자 승인 대기 결정

포지셔닝 문장은 다음 구조를 사용한다.

`[타겟]이 [상황]에서 [불안]을 겪을 때, [브랜드]는 [검증 가능한 해결 태도]를 [반복 가능한 증거 방식]으로 보여 준다.`

포지셔닝 문장에는 최종 제안명을 사용하되, 이름 상태가 `proposed`이면 문서 안에 `제안명·이미지 사용 미승인`이라고 표시한다.

## 표현 시스템

### 고정할 것

- 한글 폰트 패밀리와 굵기 단계
- 핵심 중립색·텍스트색·브랜드 강조색의 역할
- 사진의 빛, 피사체 거리, 제품 접지, 기능 매크로 방식
- 카드 모서리·그림자·선·숫자 마커
- 제목 문장 길이와 말투
- 주장보다 증거를 먼저 보이는 순서
- 브랜드 기억 장치 1~2개

### 제품별로 바꿀 것

- 카테고리 강조색 한 가지
- 사용 상황과 소품
- 기능 증거의 종류
- 페이지별 레이아웃 모듈
- 제품 고유 카피

브랜드 기억 장치는 반복 가능한 시각·언어 규칙이다. 예: 기능 증거에만 쓰는 번호 원, 특정 모서리 컷, 증거 문장 어미, 닫는 장의 3포인트 배열. 모든 페이지에 로고를 찍는 방식은 기억 장치로 세지 않는다.

## 목소리

- 과장 대신 구체적인 동작·구조·사용 상황을 말한다.
- 고객을 겁주거나 경쟁사를 비방하지 않는다.
- 제품이 보여 주지 못하는 효과를 감성어로 우회하지 않는다.
- 제목은 구매 질문에 답하고, 본문은 증거 범위를 설명한다.
- 같은 추상어와 문장 꼴을 반복하지 않는다.

## 브랜드 QA

다섯 항목을 각각 0~2점으로 평가한다. 8/10 이상이고 0점이 없어야 한다.

1. 포지셔닝이 고객 불안과 연결되는가
2. 약속이 제품 증거로 반복 입증되는가
3. 말투가 10장과 후속 상품에서 재현 가능한가
4. 색·폰트·사진·컴포넌트가 하나의 시스템인가
5. 카테고리 친숙성과 고유 기억 장치가 함께 있는가

경쟁사 색과 UI를 모아 평균낸 결과, 제품마다 완전히 다른 스타일, 승인 전 로고·슬로건 노출은 실패다.

브랜드 QA와 별도로 제품별 이름이 문서·JSON·최종 보고에 일치하는지, 제안 상태의 이름이 이미지에 노출되지 않았는지도 확인한다.
