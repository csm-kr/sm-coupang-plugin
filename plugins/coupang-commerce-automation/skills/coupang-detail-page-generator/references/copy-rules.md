# 상업형 한국어 카피 규칙

## 메시지 계층

각 페이지는 다음 중 필요한 2~3단계를 사용한다.

1. 후킹 또는 구매 이유: 모바일에서 즉시 읽히는 8~24자
2. 검증 사실 설명: 1~2문장
3. 배지·라벨·숫자: `THUMB HOLE`, `2PCS`, `01`처럼 실제 구조·구성·순서만 짧게 표시

첫 장에는 세 단계가 모두 있어야 한다. 나머지 페이지는 메시지에 맞게 밀도를 바꾼다.

## 후킹 작성

- 형태를 고객이 이해하는 사용 가치로 번역하되 기능을 만들지 않는다.
- 대조, 리듬, 짧은 명령형, 질문을 활용한다.
- 서로 다른 페이지 제목이 같은 말을 바꿔 쓴 수준이면 실패다.
- 제목마다 강조 구절 `emphasis`를 하나 지정한다. 정확히 제목 안에 포함된 2~10자 구절이어야 한다.
- `문제 제시 → 제품 답변`을 연결하되, 문제를 과장하거나 제품이 해결한다고 입증되지 않은 효과를 약속하지 않는다.
- 숫자·구성·사이즈가 검증되면 추상어보다 먼저 쓴다.

예시:

- `상완부터 손등까지 이어지는 구조` → `팔은 길게, 손은 자유롭게`
- `엄지 홀이 있음` → `엄지 홀 하나로 완성되는 손등 핏`
- `두 개 구성` → `좌·우 2개, 한 쌍 구성`

## 금지

- 근거 없는 치료·완치·통증 제거·질환 개선·교정 보장·의료기기 표현
- 근거 없는 `100%`, `완벽`, `무조건`, `최고`, `1위`와 최상급
- 임의의 충격흡수율·온도 감소율·자외선 차단율·방수 등급·내구 수치
- 근거 없는 인증·특허·시험성적서·수상·판매량·리뷰·별점
- 근거 없는 항균·탈취·냄새 제거·가격·할인율·무료배송·환불 보장
- `디테일을 보다`, `선택 포인트`, `자연스럽게 더해보세요`처럼 단독으로 구매 이유가 되지 않는 일반 문구의 반복
- 의미 없는 영어와 장식용 긴 문장
- 같은 문장 구조를 3장 이상 반복
- `한눈에`, `꼼꼼하게`, `자연스럽게`, `특별한` 같은 추상어만으로 끝나는 제목
- 승인된 근거 없이 `완벽`, `필수`, `걱정 끝`, `문제 해결`, `비교 불가`처럼 결과를 단정하는 표현

`충격완화`, `기능성`, `인체공학`도 구조 설명과 증빙 범위를 함께 제시하고 치료 결과로 연결하지 않는다. Before/After는 신체 효과가 아니라 `불편한 사용 상황 / 제품을 적용한 사용 상황`만 비교한다.

## overlay-copy.md

```markdown
## 01
- mode: conditioned
- commercial_job: hook
- brand_role: 브랜드 첫인상
- claim_id: CLAIM-01
- headline: 팔은 길게, 손은 자유롭게
- emphasis: 손은 자유롭게
- headline_alignment: center
- proof_type: worn_silhouette
- proof_frame: 실제 제품 전체 실루엣
- asset_mode: COMPOSITE
- subcopy: 상완부터 손등까지 이어지는 화이트 롱 팔토시
- badge: THUMB HOLE · 2PCS
- safe_area: top
- evidence: shape, hand_opening, components
```

`mode` 기본값은 `conditioned`다. 1차 무문자 베이스를 조건 이미지로 다시 입력하고, ImageGen은 승인 문구를 글자 단위로 그대로 렌더링한다. 모델은 문구를 새로 쓰지 않고 이미지에 맞는 배치·줄바꿈·크기·색 대비만 결정한다. 한글 오류는 해당 영역만 총 3회까지 ImageGen으로 부분 편집하며 로컬 타이포그래피 폴백은 사용하지 않는다.

초기 생성 카피가 상품명만 바꾼 일반 문구라면 `copy_status: draft_requires_fact_rewrite`로 남긴다. 제품 사실에 맞춘 고유 구매 이유와 증거가 연결되기 전에는 제작 단계로 넘기지 않는다.

버전 5 프로젝트는 모든 페이지에 `brand_role`, `claim_id`, `proof_frame`, `asset_mode`를 둔다. `asset_mode`는 `asset-strategy.md`의 해당 제품 요소와 일치해야 하며, `generated_master`는 실제 제품 증거의 출처가 될 수 없다.
