# 상세페이지 단계 지침

이 디렉터리는 선택된 한 SKU의 사실 잠금, 조사, 상품·콘텐츠기획 사용자 승인, 시각 소재, 하이브리드 HTML과 2층 QA를 담당하는 원본 스킬이다.

## 진입 게이트

- 사용자가 소싱 후보와 가격안을 승인했는지 확인한다.
- 공급처 샘플·SKU 동일성, 실제 제품 사실, 상표·이미지 권리와 필요한 규제를 확인한다.
- 확인되지 않은 정보는 `UNKNOWN` 또는 가설로 유지하고 광고 주장으로 승격하지 않는다.

## TDD와 검증 도구 우선

- 스크립트 변경 전 `../tests/detail_page/`에 실패 테스트를 먼저 추가한다.
- 검증 픽스처가 없으면 대표 프로젝트 픽스처를 먼저 만든다.
- 제품 사실, 분리 기획·승인 해시, 소재 manifest, 자산 계보, 소재 QA와 통합 QA를 기계적으로 재현한다.
- 완료 전 `python ../scripts/tdd.py verify detail-page`를 실행한다.

## 변경 금지선

- 제품 외형·색상·구성·사용 방식을 미적으로 임의 변경하지 않는다.
- 실제 증명이 필요한 장면을 생성 이미지로 대체하지 않는다.
- 상품기획과 콘텐츠기획의 현재 파일 해시를 사용자가 각각 승인하기 전 다음 단계로 진행하지 않는다.
- 테스트를 없애거나 기대값을 약화해 생성 결과를 통과시키지 않는다.
- 자동 QA와 육안 QA가 모두 통과하기 전 최종 산출물로 승격하지 않는다.
