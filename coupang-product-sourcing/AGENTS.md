# 소싱 단계 지침

이 디렉터리는 1단계 소싱의 원본 스킬이다. 플러그인 사본보다 이 경로가 정본이다.

## 반드시 읽기

1. `SKILL.md`
2. `../docs/SOURCING-PROCESS.md`
3. `../docs/SOURCING-EXECUTION-GUIDE.md`
4. 관련 `references/`와 기존 테스트
5. `../docs/RULE.md`와 관련 `../docs/ISSUE.md`

## TDD

- 구현 변경 전에 `tests/test_*.py`에 실패 테스트를 먼저 추가한다.
- 가격·마진 정책 변경은 정상가와 10% 하락 스트레스 시나리오를 모두 테스트한다.
- 반복 순회 변경은 카테고리 커서, 중복 제거, 목표 5개, 차단·재개 상태를 테스트한다.
- 보고서 변경은 도매 URL, 쿠팡 근거 5개, 마진, 로켓 수, 탈락 사유와 상태를 테스트한다.
- 완료 전 `python ../scripts/tdd.py verify sourcing`을 실행한다.

## 변경 금지선

- 미확인 비용을 0원으로 만들지 않는다.
- 공급처 MOQ·구매단위·배송비를 목록의 단가나 기본값으로 대체하지 않는다.
- 정상가·할인 참고가를 현재 실판매가로 사용하지 않는다. 의미가 확인된 가격 노드와 같은 판매 묶음 수량만 비교한다. 가격 중앙값에는 최근 구매 수 1건 이상 또는 같은 판매상품 리뷰 5개 이상인 판매 근거 행만 포함하고, 근거 없는 등록가는 제외한다. 판매 근거 가격이 5건 미만이면 가격 검토를 차단한다.
- 쿠팡 결과 0개를 수요 검증 완료로 처리하지 않는다.
- 일반 로켓 상위 10개 4개 이상을 통과시키지 않는다.
- 판매자로켓을 일반 로켓으로 잘못 분류하지 않는다.
- 통과 후보가 5개 미만인데 결과를 부풀리거나 사용자 승인 없이 상세페이지로 넘기지 않는다.
- CAPTCHA·로그인·접근 통제를 우회하지 않는다.

최종 보고서는 파일을 쓰기 전에 `python ../scripts/tdd.py report-path <run-name> --date YYYY-MM-DD --create`로 경로를 준비한다. 날짜별 현재 실행은 `../reports/YYYY/YYYY-MM-DD/<run-name>/`에 1개만 두고 이전 실행은 `../reports/deprecated/YYYY/YYYY-MM-DD/<run-name>/`에 자동 보관한다. 브라우저 수집이 끝나면 탭과 세션을 종료한다.
