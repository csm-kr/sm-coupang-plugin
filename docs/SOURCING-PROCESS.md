# 쿠팡 상품 소싱 프로세스

> 문서 탐색: [현재 상태](../STATUS.md) · [README](README.md) · [PRD](PRD.md) · [실행 가이드](SOURCING-EXECUTION-GUIDE.md) · [HDB-1 제품기획 기록](../reports/deprecated/2026/2026-07-16/hdb1-product-planning-phase1/product-plan-draft.md)

## 플러그인의 첫 번째 단계

전체 자동화는 반드시 **1단계 소싱**에서 시작한다. 소싱은 “잘 팔릴 것 같은 상품”을 고르는 일이 아니라, 공급처 원가와 쿠팡 시장 근거를 같은 후보에 연결하고 비용·마진·수요·경쟁 기준을 통과한 선택지를 만드는 과정이다. 사용자가 보고서에서 상품과 가격안을 승인하기 전에는 샘플 발주, 브랜드 확정, 상세페이지 생성으로 넘어가지 않는다.

## 목표

도매꾹 Best를 후보 풀로 사용하되 순위가 높은 상품을 자동 선택하지 않는다. 수요·마진·경쟁·운영 안전 기준을 모두 통과한 상품을 최소 5개 찾아 도매 공급처 URL과 쿠팡 근거 URL이 포함된 보고서로 제공한다. 사용자가 상품과 가격안을 선택한 뒤에만 상세페이지 제작을 시작한다.

## 도매꾹 Best 고배수 탐색 프로필

`coupang-best-high-markup-sourcing`은 일반 소싱 전에 조사 우선순위를 좁히는 전용 프로필이다. Best 전체·카테고리 TOP 150을 같은 층화 방식으로 샘플링하고 다음 조건을 모두 만족한 행을 `HIGH_MARKUP_DISCOVERY`로 둔다.

- 공급처 상세 원문의 개당 단가 5,000원 이하
- 공급처 판매 묶음과 쿠팡 비교상품이 모두 1개 구성
- 이미지·구조·규격·모델·고유 문구로 완전 동일성을 확인
- 쿠팡 할인 후 현재 실판매가가 공급 단가의 4배 이상
- 같은 쿠팡 판매상품 리뷰 5개 이상

리뷰 수는 구매 발생의 대리 신호이며 최근 수요나 구매자 수의 확정값이 아니다. 이 프로필은 정상가·취소선 가격, 다른 묶음, 유사상품 가격을 사용하지 않는다. `HIGH_MARKUP_DISCOVERY`는 높은 판매가가 관찰된 탐색 일치일 뿐 도매 배송비·물류비·수수료·부가세를 포함한 이익, 시장 가격 수용성, 경쟁과 운영 안전을 보장하지 않으므로 일반 소싱의 전체 게이트와 사용자 승인을 거치기 전 `SHORTLIST`로 승격하지 않는다.

## 전체 흐름

```text
1. 풀 구축
   전체 + 6개 대분류
   국내산 + 국외산 + 원산지 미확인
   카테고리 TOP 150 + 도전! 베스트

2. 층화 샘플링
   순위 상·중·하 × 카테고리 × 원산지 × 풀 유형
   기본 30개, 필요하면 확대

3. 저비용 사전 필터
   원문 단가·MOQ·구매단위·주문 배송비·판매/매입 수량·옵션·규제·이미지 사용권
   미확인 비용은 0원이 아니라 UNKNOWN

4. 쿠팡 검증
   카테고리·소구·동일상품 검색
   공급처 이미지·구조·규격·모델·고유 문구로 동일성 선확정
   할인 후 현재 실판매가·구성·광고·리뷰·최근 구매·판매자 분산
   동일·거의 동일·유사상품 분리

5. 가격·마진 검증
   원가 + 도매 배송 + 로켓그로스 + 수수료 + 부가세
   표준: 정상가 40% 이상 + 판매가 10% 하락 후 30% 이상
   조건부: 표준 미달, 정상가 35% 이상 + 판매가 10% 하락 후 25% 이상

6. 종합 판정
   수요, 가격 경쟁력, 동일상품 포화, 운영·규제 위험
   상세페이지에서 증명할 차별화 여백

7. 기준 통과 보고서
   통과 상품 최소 5개
   공급처 URL + 쿠팡 URL + 추천가 + 정상/스트레스 마진
   상태: AWAITING_USER_SELECTION

8. 사용자 선택
   entry / recommended / premium / reject
   선택 상품만 SHORTLIST로 승격

9. 상세페이지 핸드오프
   샘플·SKU 동일성 확인 후 상세페이지 제작 시작
```

## 도구 분담과 안전 운영

| 대상 | 표준 도구 | 운영 방식 |
|---|---|---|
| 도매꾹 Best | Browser Use | 카테고리·순위·원산지를 분산 표본화하고 상품 URL·공급가를 수집한 뒤 조사 탭을 닫음 |
| 쿠팡 | `nodriver` + 실제 Chrome | `headless=False`, 임시 프로필, 홈 선진입, 판매량순 검색 직렬 처리, 수집 후 브라우저 종료 |
| 공급·가격 계약 | `evidence_contract.py` | 공급처 원문 조건과 쿠팡 가격 의미·동일 묶음을 검증하고 모호하면 차단 |
| 마진 계산 | 로켓그로스 계산 스크립트 | 검증된 판매 묶음 원가와 단순화된 VAT 계산 방식 적용 |
| 보고 | HTML 생성기 | 통과·탈락 근거와 링크를 보존하고 `AWAITING_USER_SELECTION`에서 중단 |

쿠팡은 Browser Use 세션에서 Akamai 접근 거절이 발생할 수 있어 현재 수집 경로에서 제외했다. `nodriver`도 접근 제한을 무력화하는 우회 도구로 사용하지 않는다. 홈 진입 후 검색 결과가 0개이면 잠시 기다려 한 번만 재시도하고, 다시 0개면 `BLOCKED`와 재개 상태를 저장한다. CAPTCHA 또는 로그인 요구는 자동 처리하지 않는다.

### 브라우저·탭 종료 의무

Browser Use와 `nodriver`를 이용한 확인은 증거 저장과 브라우저 정리까지가 한 작업이다. 정상 완료뿐 아니라 접근 차단, 빈 결과, 로그인·CAPTCHA, 타임아웃과 예외 발생 시에도 정리한다.

- 브라우저를 상품마다 새로 띄우고 닫지 않는다. 하나의 제한된 조사 회차에서는 같은 정상 세션을 유지해 대상 상품을 직렬 확인하고, 회차가 끝날 때 한 번 정리한다.
- Browser Use는 현재 실행이 연 탭의 target ID를 기록하고, 확인이 끝나는 즉시 그 탭만 닫는다. 사용자가 원래 열어 둔 탭과 다른 Codex 세션의 탭은 닫지 않는다.
- Browser Use 원격 브라우저를 시작한 경우에는 열린 탭 정리 후 해당 원격 daemon도 명시적으로 종료한다.
- `nodriver`는 상품마다 새 프로필을 만들지 않고 회차마다 별도 임시 Chrome 하나를 띄워 후보를 직렬 처리한다. 회차 종료 시 `try/finally`의 `finally`에서 `browser.stop()`을 호출한다. 현재 실행이 띄운 Chrome·탭만 종료하며 사용자의 기존 Chrome은 종료하지 않는다.
- 종료 실패를 숨기지 않는다. 실행 보고서에 `browser_cleanup_status: PASS | FAIL`, 열린/닫힌 탭 수, 대상 세션 또는 프로세스, 종료 시각과 오류를 기록한다.
- 브라우저 정리가 확인되지 않은 실행은 `cleanup_status=FAIL`로 남기며 정상 완료로 보고하지 않는다.

## 현재 마진 입력 원칙

| 항목 | 값 |
|---|---:|
| 공급가·MOQ·구매단위 | 공급처 원문 검증값 필수 |
| 도매 배송비 배분 | 주문 배송비 ÷ 실제 매입 수량 × 판매 묶음 수량 |
| 쿠팡 비교가격 | 할인 후 현재 실판매가 + 동일 판매 묶음 + 판매 근거만 |
| 판매 근거 | 최근 구매 수 1건 이상 또는 리뷰 5개 이상(`review_proxy`) |
| 최소 가격 표본 | 판매 근거가 있는 독립 상품군 5개 이상 |
| 로켓그로스 비용 | 3,000원/개 |
| 판매 수수료 | 10.8% |
| 표준 목표 마진 | 정상가 40% 이상, 10% 하락 시 30% 이상 |
| 조건부 목표 마진 | 표준 미달, 정상가 35% 이상, 10% 하락 시 25% 이상 |

공급조건과 쿠팡 실판매가는 가정할 수 없다. 원문 검증이 끝난 값만 자격 판정에 사용한다. 로켓그로스 비용과 수수료는 탐색 시나리오이며 사용자가 후보를 선택하면 실제 크기·무게·입수량·카테고리 수수료를 확인해 다시 계산한다. 광고비는 초기 소싱 마진에 넣지 않고 판매 반응이 확인된 상품의 확장 판단에서 별도로 다룬다.

가격을 비교하기 전에 제품 동일성을 잠근다. 공급처와 쿠팡의 이미지·구조·규격·모델·고유 문구가 모두 맞고 판매 묶음도 같을 때만 그 현재가를 직접 가격 제약으로 사용한다. 그중 최근 구매 수 1건 이상 또는 리뷰 5개 이상인 가격만 중앙값에 넣고, 판매 근거 없는 등록가는 제외한다. 리뷰는 구매 발생 대리 신호이며 현재 가격 판매량 확정값이 아니다. 판매 근거 가격이 5건 미만이면 등록가 전체 중앙값이나 고가 한 건으로 보완하지 않고 가격 검토를 차단한다. 다른 상품 가격은 시장 맥락으로 남기되 단독 탈락 근거로 쓰지 않는다. 표준 40%/30%를 충족하지 못해도 35%/25%를 충족하면 `CONDITIONAL_TEST_PRICE_REVIEW`로 상시 분류한다. 조건부 후보는 보고서에 포함하되 표준 통과와 자동 `SHORTLIST`·핸드오프에 합치지 않는다.

## 탐색 비율

| 축 | 기본 배분 |
|---|---|
| 순위 | 1~30위 40%, 31~75위 35%, 76~150위 25% |
| 카테고리 | 6개 대분류 균등 목표 |
| 원산지 | 국내 45%, 국외 45%, 미확인 10% |
| 풀 유형 | 카테고리 TOP 150 80%, 도전! 베스트 20% |

풀이 비어 있으면 해당 몫을 존재하는 층으로 재배분한다. 이 비율은 탐색 다양성을 위한 것이며 최종 합격 수를 인위적으로 맞추는 기준이 아니다.

## 완료 기준

- 기준 통과 상품 5개 이상
- 모든 상품에 실제 공급처 URL과 쿠팡 근거 URL 존재
- 표준 40%/30% 또는 조건부 35%/25% 마진 검증과 등급 표시
- 쿠팡 수요 검증 완료
- 운영·규제 안전 검증 완료
- 미확인 비용과 동일성 충돌 없음
- 공급처 단가·MOQ·구매단위·배송비와 판매 묶음 수량 검증 완료
- 쿠팡 할인 후 현재 실판매가·동일 묶음·판매 근거 가격 표본 5건 이상 검증 완료
- 사용자 선택 전 상세페이지 단계 진입 금지

5개를 채우지 못하면 결과를 부풀리지 않는다. 카테고리·원산지·순위 구간 또는 도매매와 향후 공급처 어댑터로 풀을 확장하고 다시 검증한다.

## 반복 조사와 로켓 판정

누적 통과 상품이 5개보다 적으면 다음 카테고리·순위 구간·원산지 표본으로 자동 확장한다. 실행기는 카테고리 커서와 조사 상품 ID를 저장해 회차 간 중복을 막는다. 최대 회차, 풀 소진, CAPTCHA, 로그인 요구, 사이트 차단 또는 판매자 확인 불가는 안전 중단 조건이다.

쿠팡의 `판매자로켓`은 외부 로켓그로스 셀러이므로 진입 가능하다. 판매량순 상위 10개에서 일반 로켓이 3개 이하면 진입 가능 구간, 4개 이상이면 경쟁 과다 탈락으로 판정한다.

각 후보의 쿠팡 근거는 한 상품에 의존하지 않는다. 판매량순 상위 10개를 수집하고, 이름·구성 유사도와 가격 노드의 의미를 적용해 할인 후 현재 실판매가가 확인된 동일 묶음 중 최근 구매 1건 이상 또는 같은 판매상품 리뷰 5개 이상인 가격 표본을 최소 5개 확보한다. 정상가·취소선 가격, 의미가 모호한 복수 가격 또는 판매 근거가 없는 가격은 분포에서 제외하고 각 제외 수량을 기록한다. HTML에는 적어도 5개의 쿠팡 상품 URL 또는 그 검색 키워드 근거, 추천가, 정상·스트레스 마진, 일반 로켓 수, 판매자로켓 수와 탈락 사유를 표시한다.

## 판정 상태

| 상태 | 의미 |
|---|---|
| `SHORTLIST` | 필수 기준을 통과해 사용자 선택 대상으로 제시 |
| `CONDITIONAL` | 35%/25% 조건부 마진을 통과했지만 사용자 승인 전 자동 승격 금지 |
| `WATCH` | 가능성은 있으나 비용·수요·동일성 등 핵심 근거 부족 |
