# 문서 지침

`docs/`는 제품·아키텍처·프로세스의 정본이다. 문서가 충돌하면 `ADR` → `PRD` → `ROADMAP` → 구현 계획 → 세부 프로세스 순으로 판단한다.

- 현재 결과가 바뀌면 `../STATUS.md`와 `ROADMAP.md`를 함께 갱신한다.
- 장기 결정은 기존 ADR을 덮어쓰지 말고 새 ADR로 추가한다.
- 실행 명령은 실제 CLI `--help`와 테스트 결과를 확인한 뒤 적는다.
- 보고서 링크는 `../reports/YYYY/YYYY-MM-DD/<run-name>/` 구조를 사용한다.
- 코드와 문서가 다르면 차이를 숨기지 말고 현재 구현 상태를 명시한다.
- 문서만 바꿀 때는 코드 테스트가 필수는 아니지만 `python ../scripts/tdd.py check-routing`과 링크·경로 검사를 실행한다.
