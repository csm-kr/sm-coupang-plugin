2026-07-20에는 사용자의 병행 작업을 방해하던 브라우저 포커스 의존성을 제거했다. 도매꾹은 별도 임시 프로필의 로컬 headless Chrome에 Browser Harness를 연결하고, 쿠팡은 `headless=True`의 임시 `nodriver` 세션을 사용한다. 한글 PowerShell 표준입력을 UTF-8로 정규화한 실제 Browser Harness 스모크에서 `example.com` 연결·4프레임 기록·소유 프로세스 정리를 확인했고 최신 소싱 테스트 54개가 통과했다. 이 전환은 런타임 구현 완료이며 팔토시·깔창의 실제 원문·현재가 재수집을 대신하지 않으므로 기존 `PRICE_REVIEW_BLOCKED`는 유지한다.

같은 날 `paltosi-001`에 전체·개당 5,000원 이하·최소 3배 입력으로 실제 headless 재수집을 2차 실행했다. 도매꾹은 기본 임시 경로에서 Chrome이 시작 중 종료됐고, 워크스페이스 임시 프로필에서는 Chrome CDP 준비 뒤 Browser Harness daemon이 loopback CDP에 연결하지 못했다. headless `nodriver`는 기본 경로에서 브라우저 생성 후 쿠팡 홈 진입 전 CDP 연결이 닫혔고 명시적 Chrome 경로 재시도도 연결에 실패했다. 사용자 Chrome, 표시형 브라우저, 유료 원격 브라우저와 `no_sandbox` 우회는 사용하지 않았다. 2026-07-17 보존 후보 10건은 현재값으로 재표현하지 않고 역사 공급계약 구조와 미검증값 차단형 3배 필터만 다시 실행했으며, 현재 완전 동일 1개 구성의 판매 근거 pair 0건·`HIGH_MARKUP_DISCOVERY` 0건으로 `RESEARCH_EXPANSION_REQUIRED`와 `PRICE_REVIEW_BLOCKED`를 유지했다. [차단 HTML](../reports/2026/2026-07-20/paltosi-001-high-markup-sourcing-runtime-blocked-r2/high-markup-report.html)과 [JSON](../reports/2026/2026-07-20/paltosi-001-high-markup-sourcing-runtime-blocked-r2/high-markup-report.json)을 프로젝트에 연결했다.

같은 날 고배수 소싱의 사용자 목표를 ADR-022로 다시 잠갔다. 입력은 선택 카테고리·도매꾹 개당 공급가 상한·최소 가격 배수이며, 새 프로젝트 기본값은 `전체`·`5,000원`·`3배`다. 판매 근거가 있는 동일 1개 구성 쿠팡 상품이 지정 배수 이상으로 판매되는 pair가 하나라도 있으면 탐색 일치다. 리뷰 1개 이상 또는 만족 인원 100명 이상 라벨을 판매 흔적의 대리 신호로 쓰고, 더 싼 다른 등록은 발견 pair의 탈락 근거로 사용하지 않는다. React 입력·pair 비교 화면, 필터 JSON·HTML, 플러그인 사본을 갱신해 소싱 54개·플러그인 36개 테스트를 통과했고 800px·360px 브라우저 QA에서 가로 넘침 0을 확인했다. `HIGH_MARKUP_DISCOVERY`의 전체 가격 승인·`SHORTLIST` 비승격 게이트는 유지한다.

같은 날 실제 UI 실행 중지에서 Windows npm `codex.CMD` 래퍼만 종료되고 하위 `node.exe`·`codex.exe`가 출력 파이프를 유지해 `중지 중`에 고착되는 문제를 확인했다. 고착 실행의 해당 프로세스 트리를 정리해 화면을 `중지됨`으로 복구했고, 이후 중지는 서버가 시작한 실행 래퍼 PID 기준으로 Windows 프로세스 트리 전체만 종료하도록 수정했다.

소싱 진행 화면의 정적 Acceptance Criteria와 수기 차단 사유 입력이 실제 결과처럼 보이고 내부 코드까지 노출되던 UX도 제거했다. 실행 중 결과 영역은 `진행중입니다.` 한 문장만 표시하고, 완료 뒤 최신 JSON의 완전한 도매꾹↔쿠팡 pair와 HTML 보고서만 표시한다. 실제 실행을 유지한 채 프런트만 교체한 Browser Harness QA에서 이전 안내와 `현재 차단 사유`가 사라지고 새 진행 문구가 표시됨을 확인했다.

실패 결과는 숨기지 않고 실행 오류 이벤트에서 사람이 읽을 수 있는 실패 이유를 표시한다. 실행이 성공 코드로 끝나도 보고서가 차단·검증 미완료이거나 pair가 0건이면 보고서 원인과 `candidates`·`sampled_items`의 현재 도매꾹 확인 샘플을 함께 보여준다. 보고서 계약에도 실패·차단 시 표본 상품명·URL·카테고리·순위·원문 확인 단가 보존을 추가했고, 이전 실행의 샘플은 현재 실패 결과에 섞지 않는다.

> **Phase 1**이 현재 위치다. 사용자는 HDB-1 스포츠 마스크 1개·9,900원안, 제품명 `숨트임`, 1차 제품기획과 콘텐츠기획을 승인했다. `concept_only` 분기는 합성 모델 2장, 10모듈 시각 스토리보드, HTML·GIF와 수치형 통합 QA까지 통과했지만 실제 SKU 근거나 판매용 결과가 아니다. 판매용 분기는 실제 SKU 단독 원본이 없어 `BLOCKED_IDENTITY_ASSETS`다. 자산 기반 콘텐츠기획 v2의 해시 승인, 사람 없는 실제 SKU 정·후·측면·타공·4색·라벨·실측과 간이 실험이 남아 있다. 제품명과 브랜드명은 분리하며 `에어베일`은 미승인이다.

Codex 초보자용 `coupang-workflow-ui`가 추가되어 프로젝트 생성·목록·현재 단계·필수 입력·승인·차단을 React 로컬 작업창에서 관리한다. 소싱 JSON 후보를 카드로 비교해 `SHORTLIST` 상품과 가격을 확정하면 저장·승인·상품기획 이동을 한 번에 처리한다. VS Code형 프로젝트 탐색기는 실제 파일과 연결 보고서를 보여주고 HTML·이미지를 인라인 미리보기하며 PNG·JPG·GIF·WEBP 드래그앤드롭을 원본 자산 폴더에 저장한다. `Codex 작업 시작`은 현재 단계의 전문 스킬을 `workspace-write`로 자동 실행하고 JSONL 진행 이벤트를 읽기 전용 내장 콘솔에 표시하며 중지를 지원한다.

상품기획은 UI 밖에서 공급처 상세 URL 하나만으로 시작한다. `coupang-product-planning`이 상세 본문·옵션·상품정보·공개 후기·문의를 조사해 색상·사이즈·구성·소재·관리법을 먼저 찾고, 쿠팡 경쟁상품 5개 이상과 별점 1~3점 리뷰를 직접 조사한다. 조사 결과와 소구 후보를 먼저 보고한 뒤 실제 판매 SKU·타깃·판매가·묶음·판매 옵션을 사용자와 정하는 오퍼 결정 게이트를 두며, 실수령 대조가 필요한 사실만 사진·라벨·실측으로 요청한다. 현재 UI의 선입력 폼은 이 검색 우선 계약과 아직 정렬되지 않았다.

2026-07-20 도매꾹 상품 `60851997`로 검색 우선 흐름을 실제 점검해 단가·MOQ·옵션 6개·소재·원산지·공개 후기와 상세 이미지 사양을 확보하고 쿠팡 검색 결과를 수집했다. 다만 격리 headless 실행이 기존 Browser Harness 데몬의 사용자 Chrome 탭을 재사용해 조사 탭이 남는 2회차 오류가 발생했으므로 탭을 즉시 정리하고, 데몬 격리 회귀 수정 전 추가 경쟁상품 상세·저별점 리뷰 조사를 중단했다.

2026-07-19 소싱 UX는 고배수 탐색 단일 모드로 정리했다. 카테고리는 도매꾹 실제 `전체`+6개 대분류 중 선택할 수 있고 미선택 시 전체 순환한다. 버튼 실행은 실제 제품 또는 차단 결과의 HTML·JSON 보고서를 요구하며, 고배수 보고서는 판매 근거 현재가·수익률 최저~최고와 최고가 판매 근거를 표시한다. 같은 날 실사용 점검에서 종료 코드 0인데 신규 보고서가 없는 실행이 `완료`로 보인 문제를 확인해, 신규 HTML·JSON 및 `links.reportRuns` 등록을 서버 성공 게이트로 추가하고 완료 뒤 링크 자동 갱신·열기를 구현했다.

같은 날 `paltosi-001`의 명시된 재개 지점도 실제 실행했다. 기존 공급처 후보 10개의 근거 계약은 다시 통과했지만 표시형 `nodriver` Chrome이 사이트 진입 전에 생성 연결에 실패했다. 공개 웹 보조 조사에서는 정확한 상품명 2건을 찾았으나 각각 2.65배·리뷰 50, 1.83배·리뷰 0이었고 공급처 이미지·구조·규격 동일성을 잠그지 못해 직접 가격 범위에서 제외했다. 신규 차단 HTML·JSON은 2026-07-19 실행 경로에 생성하고 프로젝트 `links.reportRuns`에 등록했다.

이어서 `project-20260719-084800-654` 깔창 프로젝트에서도 고배수 탐색을 실제 실행했다. Browser Use는 작업공간 경로 문제를 해결한 뒤 Chrome 원격 디버깅 사용자 허용 대기에서 차단됐고, 표시형 `nodriver` Chrome도 쿠팡 홈 도달 전에 생성 연결에 실패했다. 공개 웹에서 단가 5,000원 이하 깔창 후보 10건과 쿠팡 보조 관찰 5건을 보존했지만 Best 순위·구매 증분·현재가·이미지 동일성을 확인하지 못해 모두 `PRICE_REVIEW_BLOCKED`로 두었다. 리뷰 24건인 정확한 제목의 1개·1,550원 행은 3.69배로 4배에 미달했고, 더 높은 2,100원·4,720원·5,710원 행은 리뷰 0건 또는 현재가·동일성 미검증이라 가격 상단에서 제외했다. 신규 HTML·JSON을 날짜 경로에 생성하고 프로젝트 `links.reportRuns`에 등록했다.

2026-07-20에는 `paltosi-001`의 같은 재개 지점을 다시 실행했다. Browser Use로 도매꾹 Best 페이지 도달에는 성공했지만 두 번째 CDP 연결에서 Chrome 원격 디버깅 사용자 허용이 필요해 DOM 추출과 실행 탭 정리가 차단됐다. 표시형 `nodriver` Chrome도 브라우저 생성 연결 실패로 쿠팡 홈에 도달하지 못했다. 2026-07-17 공급조건을 현재값으로 재표현하지 않고 보존한 채 계약과 고배수 필터를 재실행해 10건 모두 `PRICE_REVIEW_BLOCKED`, `HIGH_MARKUP_DISCOVERY` 0건을 확인했으며 신규 HTML·JSON과 프로젝트 보고서 링크를 등록했다.

## 기능 트랙별 선후관계

```text
[Phase 1 · 기반 안정화]
  플러그인·여덟 스킬·React 프로젝트 UI 통합 완료
            ↓
  지금: 회귀 테스트·실제 SKU 필드 매핑
            ↓
[Phase 2 · 연결]
  공통 프로젝트 스키마 → 후보 자동 승격·중단 후 재개
            ↓
[Phase 3 · 제작·출고]
  실제 모션 제작 → HTML·채널 패키징 → 게시 전 통합 QA
            ↓
[Phase 4 · 성장]
  판매 성과 피드백 → 공급처·채널 어댑터 확장
```

확정된 일정 추정치가 없으므로 날짜 대신 의존 관계를 표시한다. 각 단계는 앞 단계의 완료 조건과 QA 통과를 기준으로 진입한다.

## 현재 스프린트 우선순위

1. 별도 임시 프로필의 로컬 headless Browser Harness로 깔창 프로젝트의 패션잡화/화장품 Best 풀과 공급처 구매 증분부터 재수집한다. 사용자의 기존 Chrome 원격 디버깅 허용은 요구하지 않으며 headless 접근이 차단되면 표시형으로 전환하지 않고 차단 보고서를 남긴다.
2. headless `nodriver`로 깔창 10개 후보와 팔토시 10개 후보의 쿠팡 현재가·리뷰·완전 동일성을 홈 선진입 후 직렬 재수집한다. 검증된 동일상품·동일 묶음 판매 근거가 확보되기 전에는 상품 선택 게이트를 열지 않는다.
3. 사용자가 11개 자산의 역할·10장 배치·06장 `원단형 귀 구멍` 표현을 포함한 자산 기반 콘텐츠기획 v2를 승인하거나 수정한다.
4. 승인 후 실제 HDB-1 제품 단독 정·후·좌우·펼침, 타공 매크로, A~E 실측, 4색 동일 구도, 포장·케어라벨 자산을 확보한다. 공급처의 실제 인물 포함 사진은 제작 입력에서 제외한다.
5. 샘플 라벨의 76/24 일치 여부, A~E 치수 허용 오차·KC 적용 범위와 착용·호흡·물빠짐·내구·넥커버 간이 실험을 확인한다.
6. 사람 없는 실제 SKU canonical source 3~5개로 동일성을 잠근 뒤 실사진·GIF·영상 소재를 제작하고 소재별 QA를 수행한다.
7. 외부 자산과 네이티브 HTML을 조립하고 360·800px 실제 문자 좌표 QA, 통합 QA와 쿠팡용 정적 렌더를 검증한다.
8. 승인 제품명 `숨트임`과 브랜드명을 분리한다. 제안 브랜드명 `에어베일`은 별도 사용자 승인받거나 무브랜드 제작을 확정한다.
9. 실제 크기·무게·입고 조건으로 로켓그로스 비용과 9,900원 마진을 재계산한다.
10. 1차 UPF50+ 시험 이미지 원본·보고서 번호·출처 문구를 QA하고, 전체 성적서·시료 매핑은 2차 심화검증에 등록한다.
11. `handoff-shortlist.json`과 `product-plan.json`의 필드 차이를 표로 확정한다.
12. 프로젝트 상태 `1.0.0`에 후보 근거·가격·승인 필드를 연결하는 자동 승격기를 구현한다.
13. 승인된 실제 SKU에서 `phase_1` 주장 상태와 5.3 하이브리드 HTML·소재 QA·통합 QA 대표 산출물을 검증한다.
