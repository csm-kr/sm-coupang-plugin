# 승격 규칙 원장

## 운영 원칙

- 작업 시작 전 이 문서와 관련 `ISSUE.md` 항목을 확인한다.
- 동일 이슈 ID의 독립적인 실제 발생이 3회 이상이면 같은 ID의 `##` 제목으로 이 문서에 강제 규칙을 추가한다.
- 3회 이상 이슈가 승격되지 않았으면 다음 작업을 진행하지 않는다.
- `python scripts/tdd.py check-issues`로 이 조건을 검사한다.

## 승격된 규칙

## SOURCING-OFFER-EVIDENCE-001

소싱 가격·마진 계산과 문서 예시를 작성하기 전에 공급처 원문에서 다음 값을 같은 조사시각 기준으로 확인하고 서로 일치시켜야 한다.

- 단가
- MOQ
- 구매 증분
- 주문 배송비와 배송비 적용 수량 구간
- 판매 묶음 수량
- 실제 매입 수량

목록 카드, 과거 보고서, 문서 예시 또는 기억값을 원문 상세페이지 확인 없이 재사용하지 않는다. 실제 매입 수량은 MOQ 이상이고 구매 증분에 맞아야 하며, 배송비는 `주문 배송비 ÷ 실제 매입 수량 × 판매 묶음 수량`으로 배분한다. 쿠팡은 제품 동일성과 판매 묶음을 먼저 잠근 뒤 의미가 확인된 할인 후 현재 실판매가만 사용한다. 값 하나라도 없거나 충돌하면 `PRICE_REVIEW_BLOCKED`로 중단하고, 공급조건 예시를 문서에 넣을 때도 동일한 검증과 회귀 테스트를 적용한다.
