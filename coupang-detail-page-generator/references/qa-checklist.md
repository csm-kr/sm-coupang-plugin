# 10장 상세페이지 QA

## 워크플로 5.3 하이브리드 QA

5.3은 텍스트가 이미지에 포함되지 않는다. 생성·촬영한 시각 자산마다 기술·제품 동일성·주장-근거·카피 정확성·시각 품질 QA를 수행하고, 모두 통과한 자산만 HTML에 조립한다. 조립 후에는 구매 흐름, 브랜드 일관성, 반응형, 접근성, 광고 표현과 채널 규격을 다시 통합 검수한다.

- 소재 QA 정본: `output/qa/material-qa.json`
- 조립 정본: `output/html/detail-page.html`, `styles.css`, `package-manifest.json`
- 통합 QA 정본: `output/qa/integration-qa.json`
- 승인 정본: `output/approvals/product-plan-approval.json`, `content-plan-approval.json`

계획·자산·HTML·CSS 중 하나가 바뀌면 연결된 해시가 오래된 상태가 되어 재승인 또는 재QA가 필요하다. 상세 규칙은 [planning-and-hybrid-contract.md](planning-and-hybrid-contract.md)를 따른다.

## 페이지별 4중 게이트

모든 페이지는 아래 네 게이트를 각각 `PASS`로 기록한다. 하나라도 `FIX` 또는 `BLOCKED`면 최종 납품에서 제외한다.

- `A 파일·레이아웃`: 정확한 장수·순번·800×2400 규격, 모바일 가독성, 안전 여백, 제품 잘림, UI 일관성
- `B 제품 Identity`: 실제 canonical source 대비 실루엣·색·부품·로고·패턴·좌우 방향·구성 수량, 10장 간 구조 일관성
- `C 텍스트`: 기대 문구와 문자 단위 일치, 오타·중복·임의 문구 없음, 브랜드명·수치·단위·화살표 정확성
- `D 정보·광고`: 승인된 Evidence Ledger 상태만 사용, 미확정 수치·가짜 가격·후기·별점·인증·의료효능·경쟁사 비방 없음

OCR은 C 게이트의 보조 신호다. 기대 문구와 다른 후보를 찾되 저신뢰·미검출은 사람이 원본을 확대해 판정한다.

## 실제 인물 0건

- 큐와 매니페스트의 `human_origin_policy`가 `synthetic_only`인가
- 모든 작업의 `real_person_pixels_allowed`가 `false`인가
- `identity_seed`, `wearing_target`, `edit_target`, `preserve_edit`, 실제 사람 누끼가 없는가
- 원본 착용자 얼굴·피부·손·팔·몸·포즈가 최종 결과에 보존되지 않았는가
- 실제 사람이 포함된 원본이 ImageGen 입력에서 완전히 제외됐는가
- 레퍼런스 인물을 닮게 만들지 않았는가
- 얼굴이 보이는 전신 합성 모델이 기본 1장·최대 2장인가
- 같은 모델의 동일 포즈·보행·전신 크롭이 다른 페이지에서 반복되지 않는가
- 얼굴 없는 손·발 시연도 새로 합성됐고 실제 사람 픽셀 입력이 없는가

하나라도 실패하면 해당 페이지는 즉시 폐기한다.

## 10장 배치

- PG-01~PG-10과 최종 01~10이 정확히 1:1 대응하는가
- 누락, 중복 파일, 같은 이미지의 크롭 재사용이 없는가
- 페이지별 중간 승인을 요구하지 않고 한 생성 세션에서 처리했는가
- PG-01 이후 인물 작업이 합성 모델 기준을 재입력했는가

## 제품 사실성

- 실제 SKU 동일성 판단이 `raw_capture` 또는 신뢰 가능한 `manufacturer_source`에 근거하는가
- `generated_master`와 `generated_scene`만으로 제품 구조를 확정하지 않았는가
- 전체 실루엣·앞코·입구·스트랩·잠금부·절개·밑창·각인·구성의 확인 상태가 기록됐는가
- 실루엣, 수량, 색, 개구부, 봉제, 라벨 위치, 표면 표현을 원본과 비교했는가
- 존재하지 않는 홀, 커프, 지퍼, 버튼, 패턴, 장식, 구성품이 추가되지 않았는가
- 제품이 장갑, 다른 의류, 다른 패키지, 다른 재질처럼 변형되지 않았는가
- 생성 라벨이 틀리게 읽히지 않는가
- 실제 라벨을 합성했다면 사람 없는 제품 전용 원본인가
- 인물 장면의 손가락·관절·팔·착용 방향이 자연스러운가
- 여러 장면과 사용 단계에서 스트랩·잠금·절개·밑창 구조가 동일한가
- 실제 소스부터 생성 결과까지 제품 자산 계보가 추적되는가
- 동일성이 불명확한 경우 생성 반복 대신 필요한 부위의 촬영을 요청했는가
- 촬영 없이 진행한 결과는 `연출용 콘셉트 이미지`이며 제품 증거로 쓰이지 않는가

## 레퍼런스 품질 반영

- 밝은 자연광과 화이트·아이보리·파우더 블루 팔레트가 일관적인가
- 큰 네이비 제목, 파란 강조, 짧은 설명의 3단계 위계가 있는가
- 히어로, 플랫레이, 카드, 원형 확대, 매크로, 모자이크, 정보표가 교차하는가
- 페이지 하단이 비어 미완성처럼 보이지 않는가
- 사진이 너무 작아 제품이 보이지 않거나 같은 장면이 반복되지 않는가
- 레퍼런스의 얼굴·문구·브랜드·제품 외형을 복제하지 않았는가

## 브랜드와 UI 시스템

- 브랜드 포지셔닝이 대표 고객 불안과 연결되는가
- 제품별 브랜드명 후보 3개 이상과 최종 제안 1개가 기록되고 최종 보고에서 제안명이 명시됐는가
- 제안명 상태와 이미지 사용 승인 상태가 문서·JSON·매니페스트에서 일치하는가
- 브랜드 약속이 최소 3개의 제품·운영 증거로 뒷받침되는가
- `working_draft`인데 이름·로고·슬로건을 최종 이미지에 만들지 않았는가
- 10장이 같은 컬러 역할·한글 패밀리·사진 원칙·카드 재질·기억 장치를 공유하는가
- 제품별 카테고리 강조색과 장면만 변주되고 브랜드 코어는 유지되는가
- `ui-guide.md`의 캔버스·안전 여백·8px 그리드·정렬 축·컴포넌트가 실제 결과에 반영됐는가
- 페이지마다 하나의 세로 중심축 또는 카드축이 일관되게 적용됐는가
- 브랜드 점수 다섯 항목이 8/10 이상이고 0점 항목이 없는가

## 상업 광고 전환력

- 첫 화면에서 질문·상황·상품 중 최소 두 가지가 1초 안에 인지되는가
- 10장이 `훅 → 문제/선택 기준 → 해결 → 증거 3종 → 사용/저항 해소 → 구매정보 → 요약`으로 이어지는가
- 페이지마다 다른 구매 이유와 그에 대응하는 제품·사용 증거가 있는가
- 비교 카드가 근거 없는 전후 효과나 경쟁상품 비방을 만들지 않는가
- 사용법·보관·관리·사이즈 정보가 제공되었다면 라이프스타일 장식보다 우선 배치됐는가
- 같은 사진 크롭, 같은 모델 포즈, 같은 중앙정렬, 같은 카드 모듈이 연속 반복되지 않는가
- 최소 5개 페이지가 접촉·구조·변형·착용·보관·실측 중 서로 다른 기능 증거를 주 비주얼로 사용하는가
- 물 접촉·굽힘·잠금·보관을 말하는 카피가 같은 페이지의 실제 제품 반응 사진과 1:1로 연결되는가
- 물방울 그래픽·화살표·아이콘만으로 기능을 대신하지 않고 제품 표면과 접촉 자체가 보이는가

## 상업형 타이포그래피

- 히어로 헤드라인이 800px 기준 74~94px, 섹션 헤드라인이 60~78px 수준의 분명한 표시 크기인가
- 헤드라인이 ExtraBold/Black, 2~3줄, 행간 0.96~1.08의 압축된 덩어리로 보이는가
- 제목 안의 `emphasis` 한 구절만 강조색이며 별도 문장으로 중복되지 않았는가
- 본문·라벨·숫자가 제목보다 분명히 작고, 밝은 배경과 컬러 면 모두 충분한 명도 대비를 확보했는가
- 비교·단계·정보 카드의 시작선, 폰트 크기, 내부 여백, 숫자 마커가 반복되는가
- 정보표의 라벨·구분점·값이 각각 고정 열을 공유하고, 두 줄 값의 둘째 줄도 같은 값 시작축에 맞는가
- 헤드라인 글리프가 세로로 늘어나거나 좁아 보이지 않고, 1~2줄 우선·3줄 이내의 넓고 안정적인 덩어리인가
- 영문 배지가 장식처럼 남발되지 않고 검증 정보만 담는가

다섯 상업성 항목 `첫 화면 훅 / 구매 이유 차별성 / 주장-증거 연결 / 타이포 위계 / 전환 흐름`을 각각 0~2점으로 기록한다. 총점 8점 미만이거나 0점 항목이 하나라도 있으면 실패다.

## 카피와 정보

- 모든 카피가 `CONFIRMED_USER`, `CONFIRMED_SOURCE`, `OBSERVED_IMAGE` 사실 또는 비주장성 사용 맥락에 연결되는가
- 사진만 보고 소재·기능·성능·수치·인증을 만들지 않았는가
- 가격, 할인, 후기, 별점, 판매량, 순위를 만들지 않았는가
- 라이프스타일 이미지를 기능 근거로 해석하지 않았는가
- 1차 생성 베이스에는 문자가 없고, 최종 조건부 타이포에는 승인된 문구만 정확히 있는가
- 한글 자모 깨짐, 오탈자, 중복, 누락, 의미 없는 추가 문자가 없는가
- 조건부 편집이 얼굴·손·제품 형태·패널·조명·배경을 바꾸지 않았는가

## 실패별 최소 수정

- 한글 오타: 해당 텍스트 영역만 ImageGen으로 총 3회까지 부분 편집
- 제품 부품·로고 누락: 실제 canonical source를 추가해 제품 영역만 편집
- 제품 전체 변형: 해당 페이지만 source routing을 다시 선택해 재생성
- 레이아웃 불일치: 제품을 보존하고 배경·카드·여백만 수정
- 허위 정보: 문구·아이콘을 제거하고 D 게이트 재검수
- 규격 오류: 800×2400 캔버스로 조립하고 잘림을 재검수

모든 시도는 `regeneration-log.json`에 실패 유형·타깃 영역·참조 source ID·결과를 남긴다. 제품·레이아웃은 2회 후 중단한다. 한글은 3회 후에도 실패하면 `BLOCKED_TEXT`로 처리하고 최종 이미지에서 제외한다. 로컬 타이포그래피 폴백·외부 조판·텍스트 합성은 금지한다.

## 최종 판정

`passed`는 실제 소스 기반 SKU 동일성, 추적 가능한 제품 계보와 canonical routing, 정확히 10장, 워크플로 5.2의 정확한 `800×2400px` 규격, 실제 인물 픽셀 0건, 합성 모델 일관성, 제품 사실성, 무문자 1차 비주얼, 정확한 조건부 타이포, 4중 게이트, 레퍼런스 품질 반영, 근거 기반 카피, 상업성 8/10 이상, 브랜드 점수 8/10 이상이 모두 통과한 경우에만 사용한다.
