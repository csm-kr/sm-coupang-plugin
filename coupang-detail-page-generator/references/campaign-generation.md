# 10장 생성 세션

## 목적

사용자는 원본 상품 이미지와 레퍼런스를 한 번 제공한다. `제품 동일성 → 시장·브랜드·광고 기획 → UI·자산 전략 → Base → Design Edit → Local Repair → 검수`의 게이트를 지키며 내부 의존성을 처리하고 최종 10장을 한 생성 세션에서 완성한다. 페이지별 사용자 승인 요청은 하지 않지만 세 제작 버전의 계보는 유지한다.

## 제작 전 강제 게이트

- 실제 SKU 동일성은 `raw_capture` 또는 신뢰 가능한 `manufacturer_source`로 잠긴다.
- `generated_master`와 `generated_scene`은 실제 제품 근거로 사용하지 않는다.
- 앞코·입구·스트랩·잠금·절개·밑창·각인·구성 중 하나라도 불명확하면 생성을 시작하지 않고 필요한 촬영을 요청한다.
- `concept_only` 예외는 사용자가 명시적으로 선택한 경우만 허용하며 모든 결과를 `연출용 콘셉트 이미지`로 표시한다.
- 브랜드 시스템과 UI 가이드가 승인 가능한 작업 상태로 채워져 있어야 한다.
- 실제 SKU는 서로 다른 면을 보여 주는 canonical source 3~5개가 잠겨 있고, 각 페이지의 source ID가 `reference-routing.json`에 선택되어 있어야 한다.

## 두 단계 병렬 처리

### 1단계

- `PG-01`: 텍스트에서 새로 만든 합성 모델 히어로
- `PG-02`, `PG-03`, `PG-04`, `PG-05`, `PG-06`, `PG-08`, `PG-09`, `PG-10`: 사람 없는 제품·기능 증거 중심 페이지

가능하면 동시에 생성한다. `PG-01`은 실제 착용자나 레퍼런스 인물을 정체성 근거로 사용하지 않는다.

### 2단계

`PG-01`을 합성 정체성 기준으로 다시 입력하고 손·발 시연이 필요한 `PG-07`만 합성 인물 작업으로 생성한다. 얼굴이 보이는 추가 인물이 꼭 필요한 상품만 최대 한 페이지를 더 허용하며, PG-01과 다른 포즈·카메라·크롭을 사용한다.

도구가 병렬 처리를 지원하지 않으면 같은 세션에서 순차 처리한다. 중간 실패는 해당 작업만 재시도하고, 사용자에게 페이지별 승인을 요구하지 않는다.

## 금지 모드

- 실제 인물 보존 편집
- 원본 사람의 얼굴·몸·포즈 유지
- 실제 인물 누끼 또는 피부 픽셀 합성
- 레퍼런스 인물 닮은꼴 생성
- 원본 착용 사진 위 배경만 교체

모든 사람은 새로 생성한다. 제품만 있는 원본의 제품 픽셀은 증명용 후합성에 사용할 수 있지만, 사람 픽셀이 조금이라도 섞인 원본은 ImageGen 입력과 최종 합성에서 모두 제외한다.

## 10개 작업

| 작업 | 역할 | 사람 | 정체성 기준 |
|---|---|---|---|
| PG-01 | 히어로 | 합성 | 텍스트에서 신규 생성 |
| PG-02 | 제품 전체·구성 | 없음 | 해당 없음 |
| PG-03 | 접촉·고객 상황 | 없음 | 해당 없음 |
| PG-04 | 전체 실루엣·결합 구조 | 없음 | 해당 없음 |
| PG-05 | 변형·핵심 기능 | 없음 | 해당 없음 |
| PG-06 | 원단·봉제·라벨 | 없음 | 해당 없음 |
| PG-07 | 사용 순서 | 얼굴 없는 합성 손·발 | PG-01 안전 기준 |
| PG-08 | 기능 사용 상황 모자이크 | 없음 | 해당 없음 |
| PG-09 | 상품 정보 배경 | 없음 | 해당 없음 |
| PG-10 | 기능 중심 클로징 | 없음 | 해당 없음 |

## 승인 기록

각 작업은 다음을 기록한다.

- `synthetic_person_check`: 인물이 합성 모델인지
- `source_person_pixels`: 실제 사람 픽셀이 0인지
- `text_free`: 1차 생성 베이스에 문자·로고·워터마크가 없는지
- `art_direction_match`: 레퍼런스의 시각 원리는 반영했는지
- `commercial_structure`: 해당 페이지가 훅·비교·증거·단계·정보·요약 역할을 명확히 수행하는지
- `product_fidelity`: 실루엣, 개수, 색, 개구부, 봉제, 라벨, 소재 표현
- `sku_identity`: 실제 소스와 앞코·입구·스트랩·잠금·절개·밑창·각인·구성이 일치하는지
- `source_lineage`: 실제 제품 소스부터 생성 결과까지 추적되는지
- `brand_consistency`: 브랜드 컬러 역할·폰트·사진·컴포넌트·기억 장치가 지켜졌는지

하나라도 실패하면 해당 페이지만 재생성한다.

실패 유형별로 타깃 영역과 reference source ID를 `regeneration-log.json`에 남긴다. 제품·레이아웃 국소 수정은 페이지당 최대 2회, 한글 영역 ImageGen 부분 편집은 총 3회로 분리한다. 한글 오타는 텍스트 영역, 부품 누락은 제품 영역, 레이아웃 불일치는 배경·카드·여백만 수정한다. 제품 전체가 다른 모델로 변하면 해당 페이지만 source routing을 다시 고르며, 여러 장면의 구조가 달라지면 생성 반복을 중단하고 촬영 게이트로 돌아간다.

로컬 타이포그래피 폴백·외부 조판·텍스트 합성은 금지한다. 한글 부분 편집 3회가 모두 실패하면 해당 페이지를 `BLOCKED_TEXT`로 처리하고 최종 이미지 세트에서 제외한다.

승인된 무문자 베이스는 다시 생성하지 않고 2차 조건부 타이포 편집의 입력으로 사용한다. 2차 편집은 `TY-01`~`TY-10`으로 별도 기록하며, 정확한 카피 이외의 시각 요소는 보존한다.

## 세 버전

- `PG-XX` 또는 `page-XX-base`: 실제 제품·배경·레이아웃·무문자 UI 공간
- `TY-XX` 또는 `page-XX-designed`: 브랜드 UI와 승인 한글을 통합한 조건부 편집
- `page-XX-final`: 오탈자·정렬·제품 보존·브랜드 QA가 끝난 최종 게시본

한글 오류 때문에 Base를 다시 만들지 않는다. 제품 구조 오류 때문에 타이포만 고치지 않는다. 오류가 생긴 단계로만 회귀한다.

최종 게시 전 각 페이지에 `파일·레이아웃 / 제품 Identity / 텍스트 / 정보·광고` 네 판정을 별도로 기록하며 네 항목이 모두 통과해야 한다.
