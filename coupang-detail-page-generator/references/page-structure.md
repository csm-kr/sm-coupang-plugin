# 10장 페이지 구조

## 고정 역할과 전환 흐름

1. `hero / hook`: 합성 모델 또는 큰 제품으로 질문·상황·핵심 상품 정의. 얼굴이 보이는 전신 모델은 기본적으로 이 페이지 한 장에만 사용
2. `product_overview / solution_overview`: 제품 전체와 구성, 가장 강한 구조
3. `problem_context / problem_or_criterion`: 고객 고민을 만드는 접촉·행동 상황과 제품의 대응을 근거 안에서 시각화
4. `construction / proof_1`: 전체 실루엣·길이·결합 구조 증명
5. `key_detail / proof_2`: 개구부·커프·조절부 또는 굽힘·접힘 같은 핵심 기능 증명
6. `material_detail / proof_3`: 소재 반응·표면·보관·휴대 또는 구성 증명
7. `daily_use / use_or_demonstration`: 한 장면 또는 2~3단계 사용법
8. `lifestyle_mosaic / friction_reducer`: 보관·관리·휴대·옵션 또는 서로 다른 사용 장면
9. `product_information / purchase_information`: 검증된 구성·색상·사이즈·고지 정보
10. `closing / recap`: 큰 제품과 서로 다른 기능 증거 3가지를 제품 중심으로 요약

## 메시지 규칙

- 한 페이지에는 하나의 구매 이유만 둔다.
- 얼굴이 보이는 전신 합성 모델은 기본 1장, 최대 2장만 사용한다. 두 장을 쓰더라도 포즈·카메라·크롭은 완전히 달라야 한다.
- 02·03·04·05·06·08·09·10은 사람 없는 제품 증거를 우선한다. 사용 시연에 손·발이 꼭 필요하면 얼굴 없는 합성 클로즈업만 사용한다.
- 라이프스타일은 사용 분위기만 보여주며 기능·성능의 증거로 쓰지 않는다.
- 제품 구조 설명은 원본에서 직접 보이는 사실과 제품 전용 원본 픽셀만 근거로 한다.
- 정보가 없는 페이지 09는 기능을 만들지 말고 제공된 사실만 정리한다.
- `problem_context`는 근거 없는 불편·실패·효과 비교를 만들지 않는다. 실제 사용 장면 또는 검증 가능한 선택 기준만 비교한다.
- 04~06은 같은 사진의 확대 반복이 아니라 서로 다른 증거 종류를 사용한다.
- 07~08은 사용법·보관·관리 정보가 있으면 라이프스타일보다 먼저 배치한다.
- 열 장 중 최소 5장은 접촉·구조·변형·착용·보관·실측 중 서로 다른 기능 증거가 화면의 55% 이상을 차지한다.
- 카피가 물 접촉·굽힘·잠금·보관을 말하면 같은 페이지 사진 안에서 그 반응을 직접 확인할 수 있어야 한다.
- 모든 Page Spec에 `commercial_job`, `buyer_question`, `evidence_ledger_ids`, `headline`, `subcopy`, `headline_alignment`, `emphasis`, `proof_type`, `evidence`, `required_product_view`, `reference_source_ids`, `scene`을 기록한다.
- 모든 Page Spec에 `brand_role`, `claim_id`, `asset_mode`, `real_photo_required`, `real_photo_replacement`, `gif_candidate`, `forbidden_elements`, `qa_criteria`도 기록한다.
- 제품 구조를 보여 주는 페이지는 실제 SKU 소스 ID까지 추적한다. `generated_master`는 증거 ID가 될 수 없다.

## 시각 리듬

레퍼런스와 같은 밝고 연속적인 인상을 만들되 화면을 복제하지 않는다.

- 단 한 번의 풀 프레임 인물 → 제품 플랫레이 → 접촉 매크로 → 구조 확대 → 변형 연속컷 → 휴대·보관 → 사용 단계 → 상황 모듈 → 정보표 → 제품 중심 클로징 순으로 변주한다.
- 딥 네이비 제목, 명확한 강조색, 흰 카드와 고채도 구간을 일관되게 사용한다. 팔레트는 상품과 사용자 레퍼런스에 맞춘다.
- 첫 콘텐츠를 상단 140px 안에 시작한다.
- 페이지 높이의 70% 이상에 의도된 콘텐츠를 배치한다.
- 같은 레이아웃을 연속 사용하지 않는다.
- 브랜드의 컬러 역할·한글 패밀리·카드 재질·기억 장치는 반복하고, 페이지 역할의 레이아웃과 카테고리 장면만 변주한다.
- 여백은 카피 안전 영역으로 남기며 빈 페이지처럼 보이는 하단 공백은 금지한다.

## 권장 규격

- 워크플로 5.2: 폭 800px, 높이 2400px 고정
- 워크플로 5.1: 기존 매니페스트 규격 보존
- 워크플로 5.0 이하: 기존 매니페스트 규격 유지
- 사진 또는 정보 그래픽 55~85%
- 제목 8~28자, 2~3줄 이내
- 보조 문구 1~2문장
- 페이지당 배지 0~2개
- 상업성 점수 8/10 이상, 0점 항목 없음
