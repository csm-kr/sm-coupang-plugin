# 기능 중심 상세페이지 증거 설계

분위기 사진보다 제품이 무엇을 하고 왜 유용한지를 먼저 보여 준다. 각 기능 페이지는 아래 다섯 항목을 모두 가져야 한다.

1. `customer_situation`: 고객이 제품을 필요로 하는 접촉·행동·불편 상황
2. `product_action`: 제품이 감싸기·잠그기·접기·굽히기·분리하기처럼 실제로 하는 일
3. `visible_response`: 사진에서 확인할 수 있는 물방울, 굽힘, 결합, 개폐, 표면, 크기 변화
4. `proof_frame`: 매크로·연속 동작·상태 비교·단면·실측 중 하나
5. `bounded_copy`: 검증 사실 범위 안에서 사진이 보여 주는 내용만 설명하는 문구

## 기본 배분

- 얼굴이 보이는 전신 모델 장면은 기본 1장, 최대 2장만 사용한다.
- 같은 모델의 같은 포즈·보행·전신 크롭을 다른 페이지에서 반복하지 않는다.
- 열 장 중 최소 5장을 서로 다른 기능 증거 페이지로 만든다.
- 기능 증거는 최소 4종을 섞는다: 접촉, 구조, 변형, 사용 순서, 보관·휴대, 실측·옵션.
- 얼굴이 필요 없는 기능 시연은 제품 단독, 신발 폼, 소품, 얼굴 없는 합성 손·발 클로즈업을 우선한다.
- 같은 사진의 확대·축소·좌우 반전은 별도 증거로 세지 않는다.

## 카테고리별 접촉 증거

- 우천·방수용품: 제품 바깥면에 맺힌 물방울, 흐르는 물, 젖은 노면과 제품의 접촉을 보여 준다. 상품 근거가 있을 때만 `방수 커버`라는 카테고리 표현을 사용하고, `완전 방수`, `침수 없음`, 방수 등급·시간은 시험 근거 없이는 금지한다.
- 유연 소재: 한 제품이 굽힘·말림·펼침으로 이어지는 연속 상태를 보여 준다. 제품 형태가 다른 물건처럼 변형되면 실패다.
- 잠금·조절 구조: 잠금 전·맞춤·잠금 후의 손 동작과 결합부를 같은 방향에서 보여 준다.
- 휴대 제품: 접힌 크기와 보관 위치를 한 프레임 안에서 비교하되 실제 크기 수치를 만들지 않는다.
- 밑창·표면: 패턴과 접촉 위치는 보여 줄 수 있지만 시험 근거 없는 미끄럼 방지·마찰 성능을 단정하지 않는다.

## 실패 조건

- 모델 얼굴이 제품보다 더 크게 반복된다.
- 물·굽힘·잠금·보관을 말하지만 해당 반응이 사진에 보이지 않는다.
- 라이프스타일 장면만 바꾸고 같은 제품 각도와 포즈를 반복한다.
- 기능 한 가지를 여러 페이지에서 같은 증거로 재포장한다.
- 물방울 그래픽, 화살표, 아이콘만 있고 실제 제품 표면과 접촉 장면이 없다.
- 사진이 보여 주는 범위를 넘어 성능 등급·효과·비교 우위를 단정한다.
