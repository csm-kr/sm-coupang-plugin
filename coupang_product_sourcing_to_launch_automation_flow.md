# 쿠팡 상품 소싱부터 출시·운영까지 전체 자동화 플로우

## 1. 목적

이 문서는 쿠팡에서 판매할 제품을 탐색하고, 수익성과 시장성을 검증한 뒤, 제품 및 콘텐츠를 기획·제작하고 실제 상품 등록과 운영 개선까지 이어지는 전체 업무 흐름을 정의한다.

단순히 상세페이지를 자동 생성하는 것이 아니라 다음 전체 과정을 하나의 폐쇄형 자동화 시스템으로 연결하는 것이 목표다.

> 시장 탐색 → 제품 소싱 → 검증 → 제품기획 → 콘텐츠 제작 → 상품 등록 → 테스트 출시 → 성과 분석 → 개선

# 3. 단계별 상세 플로우

## 3.1 시장 탐색

도매 사이트에서 제품을 바로 찾기 전에 어떤 시장과 고객 문제를 공략할지 먼저 결정한다.

### 주요 확인 항목

- 쿠팡 내 검색 수요
- 최근 판매 추세
- 계절성
- 주요 판매 가격대
- 핵심 구매자
- 반복 구매 가능성
- 문제 해결형 상품 여부
- 콘텐츠 차별화 가능성
- 상위 상품 집중도
- 브랜드 상품 비중
- 신규 판매자의 진입 가능성

### 핵심 판단 기준

경쟁 강도가 낮은 시장보다 전체 판매량과 시장 규모가 충분한 시장을 우선한다.

경쟁이 낮다는 것은 수요가 낮다는 의미일 수도 있다. 반대로 경쟁이 강하더라도 시장 파이가 크다면 일부 점유율만 확보해도 충분한 매출을 만들 수 있다.

### 산출물

- 후보 카테고리 리스트
- 핵심 검색 키워드
- 예상 시장 규모
- 계절성 등급
- 카테고리별 진입 우선순위
- 시장 탐색 점수

## 3.3 1차 상품 필터링

후보 상품 중 사업성이 낮거나 운영 리스크가 큰 상품을 빠르게 제외한다.

이 단계는 정밀 분석이 아니라 자동 규칙 기반의 탈락 및 보류 판정을 목적으로 한다.

### 즉시 탈락 또는 보류 조건 예시

- 예상 순마진율이 기준 미달
- 예상 개당 순이익이 기준 미달
- 필수 인증 여부가 확인되지 않음
- 공급처 재고가 불안정함
- 리드타임이 지나치게 김
- 부피와 무게가 과도함
- 파손 가능성이 높음
- 반품 가능성이 높음
- 옵션이 지나치게 많음
- 공급 가격 변동이 큼
- 경쟁 제품과 차별화가 불가능함
- 이미지·영상 등 기초 소재 확보가 어려움
- 상표권·디자인권 리스크가 큼

### 자동 필터 규칙 예시

```text
예상 순마진율 < 20% → 탈락
예상 개당 순이익 < 4,000원 → 탈락
필수 인증 미확인 → 보류
리드타임 > 30일 → 감점
예상 반품률 높음 → 감점
공급처 재고 불안정 → 감점
차별화 가능성 낮음 → 감점
```

### 산출물

- 통과 상품
- 보류 상품
- 탈락 상품
- 탈락 사유
- 추가 검증 필요 항목

### 핵심 원칙

경쟁사의 반복 불만은 이미 소싱하기로 한 해당 SKU가 해결할 수 있는지를 확인하는 제품 검증 기준으로 변환한다. 별도 OEM·제품 개량 가능성을 전제로 해결했다고 판단해서는 안 된다.

특히 경쟁사 별점 1~3점 리뷰의 반복 불만을 우선 공략한다. 그 불만을 소싱하기로 한 제품 자체가 현재 사양 그대로 해결하고 샘플·실측·출처로 입증할 수 있을 때만, 해당 해결점을 핵심 소구점 후보로 올린다.

경쟁사 리뷰에서 관찰한 장점·불만은 시장 관찰이며 곧바로 우리 제품의 소구점이 아니다. `반복 불만 → 고객 구매 기준 → 우리 제품 사실 → 샘플·실측 증거 → 승인 소구점`이 연결된 경우에만 제품기획으로 승격한다.

소싱 제품이 시장의 핵심 반복 불만을 해결하지 못하면 해당 항목을 소구점에서 제외한다. 그 불만이 구매 결정과 반품에 중대한 문제라면 제품기획을 승인하지 않고 소싱 단계로 되돌린다.

## 3.8 1차 간이검증과 2차 심화검증

> 상태: 2026-07-16 사용자 2단계 구조 및 2차 진입 기준 확정 — 캠페인 시작 후 14일 이내 판매수량 2개 이상 AND ROAS 400% 이상

처음부터 모든 검사와 증빙 비용을 투입하지 않는다. 1차에서는 적은 비용으로 소구점 가설과 시장 반응을 확인하고, 판매 가능성이 확인된 제품만 2차에서 별점·후기·다수 샘플·검사서·심화 실험을 추가한다.

### 공통 최소 안전선

- 샘플 발주·결제는 정확한 SKU·옵션·수량·비용에 대한 사용자 승인 후 진행한다.
- 실제 판매 SKU와 공급처 자료·사진·샘플의 동일성을 확인한다.
- 상품 등록 전에 반드시 필요한 법률·안전·인증·표시사항 검증은 2차로 미루지 않는다.
- 확인되지 않은 수치·효능·안전·인증·비교우위는 1차 콘텐츠에서도 사실처럼 표현하지 않는다.
- 실험하지 못한 항목은 `미검증 소구 가설`로 보존하고, 검증된 소구점과 구분한다.

### 1차 — 시장반응 확인용 간이검증

목표는 완전한 품질 보증이 아니라 경쟁사 저별점 리뷰에서 찾은 문제를 소싱 SKU가 해결할 가능성과 판매 반응을 적은 비용으로 확인하는 것이다.

#### 1차 필수 작업

- 경쟁사 별점 1~3점 반복 불만과 소구점 가설 정리
- 동일 SKU·옵션·구성의 최소 사실 확인
- 가능한 범위의 간단한 사용·구조·실측·비교 실험
- 실험 조건, 관찰 결과, 사진·영상과 한계 기록
- 공급처가 제공한 공식 사양·라벨·관리법의 출처 기록
- 검증된 사실, 간이 실험으로 지지된 소구, 미검증 가설 분리
- 1차 제품기획과 콘텐츠기획에 사용할 수 있는 주장 범위 확정

#### 1차에서 실험할 수 없는 경우

실험이 불가능해도 제품기획과 콘텐츠기획으로 넘어갈 수 있다. 다만 해당 소구는 `미검증`으로 표시하며 정량 성능·효능·안전·우월성의 확정 주장에는 사용하지 않는다. 1차 콘텐츠는 확인된 제품 사실, 관찰 가능한 구조, 사용 상황, 구성·사이즈 등 검증 가능한 정보 중심으로 만든다. 향후 근거가 생기면 교체할 수 있도록 카피와 증거 소재를 분리한다.

#### 1차 판정

- 제품기획·콘텐츠기획 진행
- 제한된 주장으로 조건부 진행
- 추가 간이 실험 후 진행
- 핵심 사실 부족으로 보류
- 소싱 단계 복귀

### 판매 반응 게이트

테스트 출시 후 판매량·전환율·광고 효율·반품·문의·초기 별점·후기를 확인한다. 현재 2차 진입 기준은 캠페인 시작 후 14일 이내 `판매수량 2개 이상 AND ROAS 400% 이상`이다. 실제 수치는 사용자가 알려 준 캠페인 성과를 승인 근거로 기록한다. 기준을 충족한 제품만 2차 심화검증과 확장 투자 대상으로 승격한다.

### 2차 — 판매확인 후 심화검증

2차 목표는 판매 확대 전에 제품 품질, 핵심 소구, 반품 위험과 주장 근거를 강화하는 것이다.

- 경쟁사 최신 별점·후기와 1차 판매에서 발생한 우리 상품의 별점·후기·문의·반품 사유 통합 분석
- 동일 SKU 다수 샘플의 외관·색상·냄새·크기·무게·재질·마감·포장 편차 확인
- 경쟁사 저별점 핵심 불만을 해결하는지 반복 사용·내구·세척·착용·설치 등 심화 실험
- 동일 SKU에 적용되는 시험성적서·검사성적서·인증서·공식 사양서 수집과 적용 범위 확인
- 자체 실험의 목적, 조건, 절차, 비교 기준, 반복 횟수, 원자료, 결과와 한계 기록
- 실제 치수·허용 오차·착용 또는 호환 범위·관리법·주의사항 확정
- 소구점과 허용 주장 범위 재판정
- 제품기획·Source of Truth·콘텐츠기획·상세페이지·상품등록 정보 재승인 및 갱신

검사서·시험 결과가 다른 모델이나 옵션에 대한 것이거나 적용 범위를 확인할 수 없으면 해당 SKU의 근거로 사용하지 않는다. 자체 실험은 공인 시험이나 인증으로 표현하지 않는다.


### 2차 판정

- 판매 확대 승인
- 근거·콘텐츠 보완 후 확대
- 제한된 판매 유지
- 공급처 변경 또는 재소싱
- 판매 중단

## 3.9 제품기획

판매할 상품의 최종 스펙과 고객 가치 제안을 결정한다.

제품기획은 경쟁사 비교표와 리뷰 불만 클러스터를 필수 입력으로 받는다. 제품기획 단계에서 우리의 소구점 후보를 만들되, 실제 제품 사실·샘플 검수·실측으로 증명할 수 없는 항목은 확정 소구점으로 승인하지 않는다.

제품기획은 `무엇을 말할 것인가`만 정하는 단계가 아니다. `누구의 어떤 불만을 해결하는가`, `소싱 SKU가 현재 상태로 어떻게 해결하는가`, `무엇으로 증명하는가`, `어디까지 표현할 수 있는가`, `사이즈·사용·관리 정보를 어떻게 정확히 안내할 것인가`를 함께 잠근다.

### 필수 입력

- 현재 검증 단계: `phase_1` 또는 `phase_2`
- 경쟁사별 핵심 소구점과 판매 방식
- 별점 1~3점 및 반복 리뷰 불만 클러스터
- 반복 칭찬과 고객이 실제로 중요하게 보는 구매 기준
- 경쟁사 설명과 실사용 리뷰의 불일치
- 미충족 수요와 반품·문의 위험
- 소싱하기로 한 동일 SKU의 검증된 사실과 샘플 검수 결과
- 동일 SKU에 적용되는 검사서·시험성적서·인증서·공식 사양서
- 자체 실험·실측의 조건, 원자료, 결과와 한계

### 1. 고객과 문제 정의

- 핵심 타깃 고객
- 고객의 핵심 문제
- 사용 상황
- 구매 전 불안과 반품·문의 원인

### 2. 경쟁사 저별점 공략점

- 별점 1~3점의 반복 불만과 출처
- 불만의 빈도·최근성·심각도·구매 영향
- 제품 자체, 설명·사용법, 포장·배송, 개인 취향의 원인 분류
- 소싱 SKU가 현재 사양 그대로 해결하는 문제
- 해결하지 못하거나 추가 확인이 필요한 문제

제품 자체의 반복 문제를 소싱 SKU가 해결하면 제품 소구점 후보로 삼는다. 설명·사용법 문제는 콘텐츠 필수 정보로, 포장·배송 문제는 운영 요구사항으로 보낸다. 개인 취향이나 단발성 불만은 반복성이 확인되기 전까지 핵심 소구점으로 일반화하지 않는다.

### 3. 소구점과 어필 요소

- 핵심 소구점 최대 3개
- 핵심 소구를 보강하는 보조 어필 요소
- 각 소구가 해결하는 저별점 불만
- 우리 제품의 구체적인 해결 사실
- 경쟁사 비방 없이 우리 제품의 장점으로 전환한 표현
- 확정·보류·탈락 상태와 그 이유

### 4. 근거와 주장 범위

각 소구점은 `경쟁사 불만 → 고객 구매 기준 → 동일 SKU의 해결 사실 → 근거 → 허용 주장 → 제한·주의`의 한 줄로 연결한다.

- 시험성적서·검사성적서·인증서의 동일 SKU 적용 여부
- 공식 사양서·라벨·사용설명서의 출처
- 자체 실험의 조건·절차·원자료·반복 결과
- 실제 샘플의 실측·사진·영상 증거
- 정량 수치, 비교 표현, 기능·안전 표현의 허용 범위
- 콘텐츠에서 사용할 증거 형식: 문서 캡처, 실측 사진, 비교 실험, GIF, 영상

증거의 강도보다 강한 표현을 사용하지 않는다. 핵심 소구점에 필요한 근거가 없으면 소구를 보류하고, 제품의 구매 이유가 사라질 정도라면 제품기획 승인을 막고 소싱 단계로 되돌린다.

1차 제품기획에서는 `검증 사실`, `간이 실험 지지`, `미검증 가설`, `사용 금지`를 분리한다. 미검증 가설이 남아 있어도 제품기획과 콘텐츠기획은 진행할 수 있지만, 판매 콘텐츠에는 해당 가설을 확정 사실로 쓰지 않는다. 2차 심화검증 후 근거가 추가되거나 주장이 바뀌면 제품기획을 다시 승인한다.

### 5. 제품 사양과 오퍼

- 제품 구성
- 수량 구성
- 색상
- 패키지
- 증정품
- 제품명
- 브랜드명
- 가격 포지셔닝
- 경쟁 제품과의 차이

### 6. 사이즈·착용·호환 정보

- 실제 제품 치수와 무게
- 치수 허용 오차와 측정 위치·방법
- 옵션별 치수 차이
- 착용·설치·호환 가능 범위
- 타깃 고객이 이해할 수 있는 사이즈 선택 기준
- 실측 도식과 비교 기준
- 맞지 않을 수 있는 조건과 구매 전 확인 사항

권장 신체 범위나 호환 범위는 추정하지 않는다. 실제 측정·착용·설치 시험 또는 신뢰 가능한 공식 자료가 있을 때만 확정한다.

### 7. 사용·관리·주의 정보

- 최초 사용 전 준비
- 올바른 사용·착용·설치 순서
- 세척·세탁 방법
- 건조·보관 방법
- 사용하면 안 되는 세제·온도·행동
- 제품 수명을 떨어뜨리는 사용 조건
- 안전 주의와 사용 제한
- 공급사 확인이 필요한 미확정 관리법

관리법은 소재, 라벨, 공급사 공식 안내와 실제 샘플 시험에 근거한다. 확인되지 않은 세척·관리 방법은 임의로 만들지 않는다.

### 8. 반품·문의 예방 정보

- 고객이 오해하기 쉬운 기능과 한계
- 실제 색상·크기·재질·구성의 명확한 안내
- 옵션 선택 오류 방지
- 예상 가능한 사용 실패와 해결 방법
- 구매 전에 반드시 확인할 조건

### 제품기획 핵심 문장

> 누구의 어떤 문제를 기존 제품보다 어떤 방식으로 더 잘 해결하는가?

### 예시

> 꽉 끼는 팔토시가 불편한 30~50대 여성 운전자를 위해 여유 있는 핏과 손등 커버 구조를 제공하는 냉감 팔토시.

### 산출물

- 제품기획서
- 고객 정의
- 문제 정의
- 가치 제안
- 핵심 소구점
- 최종 제품 사양
- 가격 포지셔닝
- 차별화 전략
- 경쟁사 리뷰에서 도출한 고객 구매 기준

- 우리 제품 소구점 후보와 근거
- 확정·보류·탈락 소구점

## 3.19 판매 반응 확인 및 2차 검증 진입 판정

테스트 결과를 기반으로 판매량과 재고를 관리하고, 2차 심화검증에 투자할 제품을 선별한다.

### 2차 진입 판단

- 캠페인 시작 후 14일 이내 판매수량 2개 이상
- 같은 판정 캠페인의 ROAS 400% 이상
- 판매 후 공헌이익과 현금흐름
- 반품·취소·문의의 위험 수준
- 초기 별점과 긍정·부정 후기의 내용
- 재주문·재고 확대 가능성
- 심화검증 비용 대비 확장 가능성

두 기준은 `AND`로 판정한다. 실제 판매수량과 ROAS는 사용자가 알려 준 값을 `CONFIRMED_USER`로 기록하며 시스템이 임의로 추정하지 않는다. ROAS는 `광고 귀속 매출 ÷ 광고비 × 100`으로 계산하고, 광고비가 0원이거나 귀속 기준이 확인되지 않으면 통과로 판정하지 않는다.

기준 미달 제품은 개선 후 재테스트, 제한 판매, 소싱 복귀 또는 판매 중단으로 판정한다. 사용자가 기준을 바꾸면 기존 판정을 덮어쓰지 않고 새 정책 버전으로 다시 계산한다.

### 캠페인별 관리 계약

캠페인 성과는 서로 섞지 않고 캠페인별로 기록한다. SKU의 2차 진입 판정에는 사용자가 지정한 `decision_campaign_id` 한 건을 사용한다. 여러 캠페인을 합산하려면 사용자가 대상 캠페인과 합산 방식을 별도로 승인해야 한다.

```yaml
campaign_id:
campaign_name:
sku_id:
channel:
started_at:
observed_at:
evaluation_window_days: 14
ad_spend:
attributed_revenue:
sales_units:
roas_percent:
metric_source: CONFIRMED_USER
reported_by:
reported_at:
gate_policy_version:
gate_result: HOLD | PASS | FAIL | STOPPED
decision_reason:
```

캠페인 시작 후 14일이 지나기 전 기준을 충족하지 못한 상태는 `HOLD`, 14일 이내 두 기준을 모두 충족하면 `PASS`로 기록한다. 14일 종료 시점에도 미달이면 `FAIL`이며, 사용자가 조기 중단하면 `STOPPED`로 기록한다.

- 광고 예산
- 키워드 입찰
- ROAS
- CPC
- 전환율
- 일 판매량
- 재고

### 문제 진단 예시

- 노출이 낮음 → 키워드·광고·카테고리 문제
- 클릭률이 낮음 → 썸네일·상품명 문제
- 클릭률은 높고 전환율이 낮음 → 가격·상세페이지·리뷰 문제
- 전환율은 높지만 이익이 낮음 → 가격·광고·수수료 문제
- 반품률이 높음 → 제품 품질·사이즈·설명 문제
- 문의가 반복됨 → 상세페이지·FAQ 보완 필요
- 특정 옵션만 판매됨 → 옵션 구조 및 재고 재설계 필요
- 리뷰에서 동일 불만 반복 → 제품 개선 또는 공급처 변경 필요

### 개선 루프

```text
성과 수집
→ 문제 진단
→ 원인 분류
→ 개선안 생성
→ 우선순위 결정
→ 필요 시 2차 심화검증
→ 제품·근거·가격·콘텐츠·광고 수정
→ 재출시 또는 재테스트
→ 성과 재측정
```

# 7. 핵심 설계 원칙

- 시장 규모를 경쟁 강도보다 먼저 본다.
- 리뷰 총개수보다 최근 리뷰 증가 속도를 본다.
- 공급가와 판매가 차이만으로 수익성을 판단하지 않는다.
- 제품 실물을 확인하기 전에 최종 콘텐츠를 확정하지 않는다.
- 1차는 간이 실험과 최소 사실로 시장 반응을 확인하고, 판매 가능성이 확인된 제품에만 2차 심화검증 비용을 투입한다.
- 1차에서 실험할 수 없는 소구 가설이 있어도 제품기획·콘텐츠기획은 진행할 수 있지만, 미검증 수치·효능·우월성을 판매 주장으로 사용하지 않는다.
- 법률·안전·필수 인증·표시사항은 판매 전 필수 검증으로 유지하며 2차로 미루지 않는다.
- 경쟁사의 반복 불만을 소싱하기로 한 해당 SKU의 해결 가능성 검증 기준으로 변환한다.
- 경쟁사 별점 1~3점 리뷰의 반복 불만 중 소싱 제품이 현재 사양 그대로 해결하고 검증할 수 있는 지점을 우선 소구점 후보로 삼는다.
- 소싱 제품이 구매 결정에 중대한 반복 불만을 해결하지 못하면 제품기획 승인을 막고 소싱 단계로 되돌린다.
- 경쟁사 리뷰는 시장 관찰로 보존하고, 우리 제품 사실과 샘플 증거가 연결된 항목만 우리의 소구점으로 확정한다.
- 모든 소구점은 동일 SKU에 적용되는 검사서·시험성적서·공식 자료·재현 가능한 자체 실험·실측 중 적절한 근거와 허용 주장 범위를 가진다.
- 제품기획에서 실제 치수·측정법·호환 범위와 사용·세척·건조·보관·주의사항을 확정해 반품과 문의를 예방한다.
- 상품기획과 콘텐츠기획을 분리하고 각각 현재 버전을 사용자가 승인한다.
- 모든 콘텐츠는 단일 Source of Truth를 참조한다.
- 카피·표·FAQ는 네이티브 HTML로, 제품·연출 비주얼은 외부 이미지·GIF·영상 자산으로 분리한다.
- 각 콘텐츠 소재를 먼저 QA하고, HTML과 판매 정보를 합친 뒤 다시 통합 QA한다.
- AI 생성물은 실제 제품의 형태와 정보를 왜곡해서는 안 된다.
- 실사 검증이 필요한 기능은 실제 촬영으로 증명한다.
- QA는 콘텐츠 검수만이 아니라 상품·정책·손익·등록 검수를 포함한다.
- 상품 등록을 완료 시점으로 보지 않고 판매 성과 개선 루프까지 포함한다.
