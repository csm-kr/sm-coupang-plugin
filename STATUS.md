## 확정된 설계·구현 대기

- 경쟁사 별점 1~3점 반복 불만에서 소구 가설을 만들고 소싱 SKU의 해결 가능성과 근거를 연결하는 제품기획 원칙 확정
- 검증을 `phase_1` 간이검증과 판매 신호 통과 후 `phase_2` 심화검증으로 나누는 ADR-014 채택
- 1차에서 실험할 수 없는 소구는 미검증 가설로 기획에 보존하되 판매 주장에서는 차단
- 2차에서 자사·경쟁사 별점·후기, 다수 샘플, 검사서·시험성적서, 심화 실험, 사이즈·관리법을 확인하고 기획·콘텐츠·QA 재승인
- 2차 진입 기준은 판정 캠페인 시작 후 14일 이내 판매수량 2개 이상과 ROAS 400% 이상을 모두 충족하는 것으로 확정
- 캠페인별 실제 판매수량·광고비·귀속 매출·ROAS는 사용자가 제공하고, 판정 캠페인과 정책 버전을 분리해 관리
- 위 2단계 상태·판매 신호 게이트·재승인 자동화는 아직 구현 전
- 공통 프로젝트 폴더·상태는 구현했지만 소싱 후보의 전체 근거·가격·승인을 자동 매핑하는 `promote_shortlist.py`는 구현 대기

사용자는 아이스 쿨링 스카프를 재검증 대상으로 선택했다. 공급처 MOQ 4개와 배송비 2,500원, 쿠팡 할인 후 현재 실판매가를 적용한 결과 2개입은 6,900원에서 약 986원 적자, 3개입은 5,900원에서 약 3,701원 적자다. 현재 공급 오퍼는 `REJECT_CURRENT_SUPPLY_OFFER`이며 가격안과 상세페이지 제작 권한은 열리지 않았다.

후속으로 #8 국산면 고급 면스카프 손수건을 검증했다. 단가 1,400원, MOQ 5개, 구매 증분 1개, 주문 배송비 3,000원을 확인하고 쿠팡 현재 실판매가를 1·3·5개 구성별 20건씩 수집했다. 1·2·3·5·10개 구성 모두 40% 마진 하한이 시장가보다 높아 현재 공급 오퍼를 탈락시켰다.

재소싱에서는 61개를 시장 선별하고 17개를 심화 동일성 검토했다. 모던 체크 발매트·판초우비·무릎보호대·넥 스피릿 쿠션·분리형 햇빛차단 모자·나비 페이스 마스크 등은 완전히 같거나 거의 같은 상품의 현재가가 고정비 또는 조건부 하한보다 낮아 탈락했다. 표준 40%/30% 확정 통과는 0건이다. 상시 허용하는 조건부 35%/25% 기준에서는 HDB-1 스포츠 마스크 9,900원과 접이식 백팩 14,900원 두 건이 마진을 넘었지만, 상위 20개에서 완전 동일 쿠팡 상품을 찾지 못해 가격 수용성 미확인 상태로 분리했다.

사용자는 HDB-1 스포츠 마스크 1개·판매가 9,900원안을 `phase_1` 제품기획 대상으로 선택했다. 정상가 마진 39.83%, 10% 인하 후 35.33%이며 표준 정상가 하한에는 0.17%p 미달하는 조건부 선택이다. 락브로스 WB-003의 저별점 베스트순 30건에서 착용 압박 14건, 호흡 불편 10건, 물빠짐 4건을 우선 문제로 분류했다. HDB-1의 재질 76/24, 제조·판매 정보와 A~E 실치수 25·13.7·6.7·28.5·4.5cm를 원장에 반영했고, 실제 옵션은 블랙·다크그레이·그레이·핑크 4색으로 확인했다. 공급처 후기 35건 중 통풍·편안함을 직접 언급한 5점 후기 1건과 추가 착용감 긍정 후기를 확인했다. 사용자는 `입·코 주변 구멍이 있어 숨쉬기 편한 구조`를 핵심 효용 소구로 확정하고 제품기획을 승인했으며, 이어 10개 모듈·이미지+HTML 분리·촬영/GIF·소재 QA·통합 QA 콘텐츠기획도 승인했다. 제품명은 `숨트임`으로 승인했으며 브랜드명과 분리해 관리한다. 현재는 실제 SKU의 제품 단독 사진·타공 매크로·A~E 실측·4색·라벨 자산과 호흡·착용·세탁 간이 QA를 준비하는 단계다. 전체 Intertek 보고서·색상별 시료 매핑은 2차 심화검증으로 넘긴다.

공급처 페이지에서 자산 11개를 추가 인입했다. 최신 제품 사진 8개는 모두 실제 인물 픽셀을 포함해 ImageGen 입력과 최종 합성에서 차단했고, 별도의 Gemini 생성 치수 이미지는 제품·치수 근거에서 제외했다. 76/24 상품정보표와 Intertek `ITKD25025374` 3/7쪽은 문서 근거로 잠갔다. 실제 출고 SKU의 사람 없는 제품 단독 원본이 0개이므로 `hdb1-sumteuim-phase1` 프로젝트의 제품 동일성 상태는 `BLOCKED_IDENTITY_ASSETS`다. 제품기획·콘텐츠기획·제품명 승인은 유지된다. 실제 SKU 판매용 이미지 생성은 시작하지 않았고, 사용자가 별도 허용한 `concept_only` 프로토타입만 격리 경로에서 생성·검수했다.

11개 자산을 반영한 콘텐츠기획 v2 초안을 만들었다. Intertek 시험지 1개는 04장의 직접 증거, 상품정보표 1개는 08·09장의 HTML 데이터 원천, 인물 포함 사진 8개는 촬영 구도·부위 참고, 생성 치수 이미지 1개는 부정 QA 사례로 고정했다. 기존 승인된 10장 구매 서사는 유지하며 장별 Page Spec·네이티브 HTML 카피·UI·자산 전략을 구체화했다. 사진상 별도 귀 리본이 아니라 원단형 귀 구멍이므로 06장 표현을 안전하게 수정했고 E 4.5cm는 실제 측정 기준 확인 전 사양표 게시를 보류했다. 이 v2는 사용자 승인 대기이며, 승인되더라도 사람 없는 실제 SKU 자산 확보 전 제작은 차단된다.

사용자는 실제 제품 단독 원본이 없는 상태에서도 이미지+HTML·GIF 제작 방식의 품질을 보기 위해 가상 원본을 사용하는 1회성 예외 프로토타입을 명시적으로 허용했다. 가상 마스터와 모듈별 장면, Intertek 원본 문서, 3초 구조 초점 GIF를 외부 자산으로 두고 네이티브 HTML 카피·치수·사양을 조립한 10개 모듈을 만들었다. 360·800px에서 모듈 10개, 가로 넘침 0, 대체 텍스트 누락 0, 이미지 로드 10/10, JS 예외 0, GIF 정지→정적 대체 전환을 확인했다. 소재·통합 QA는 `prototype_pass_not_promotable`이며 실제 제품 동일성은 계속 `blocked`다.

추가 개선에서는 마네킹 중심의 앞 2개 모듈을 합성 성인 모델 컷으로 교체하고, 모듈마다 주장·근거·자산·필수 단서·크롭 안전영역을 가진 시각 스토리보드를 만들었다. 첫 측정에서 03장 피사체 가시율 80%, 10장 핵심 영역 99.1%, 800px 히어로 제목 4행 문제가 발견됐다. `object-fit`, 그리드와 제목 크기를 수정한 뒤 모듈 순서·주장-자산 연결·크롭이 모두 100%, 주장-이미지 연관성 평균 92.1/100, 타이포 오류·경고 0건으로 통과했다. 이 평가는 연관성과 레이아웃 품질이며 실제 HDB-1 동일성은 계속 차단한다. 완성 HTML과 미수행 `experiment-plan.json`을 분리했다.

HTML 줄바꿈은 별도 CDP 수집기로 다시 검사했다. 캐시된 이전 문서를 재측정하지 않도록 브라우저 캐시를 끄고 viewport마다 새 쿼리를 사용했으며, 실제 문자 Range 좌표로 360px·800px의 80개 텍스트 요소를 판정했다. 단어 중간 분리·한 글자 고아행·제목 행수 초과·겹침·잘림은 오류 0·경고 0으로 통과했다. 같은 viewport의 모듈별 캡처 20개도 보존했다.

## 확정된 실행 방식

### 도매꾹

- Browser Use로 Best 페이지와 카테고리별 TOP 150 조사
- 도매꾹 URL·공급가·MOQ·카테고리·원산지 근거 저장
- 필요한 표본을 수집한 뒤 현재 실행이 연 조사 탭만 닫고, 사용자·다른 세션의 기존 탭은 유지

### 쿠팡

- `nodriver`, `headless=False`, 회차별 새 임시 Chrome 세션
- 쿠팡 홈 선진입 후 판매량순 검색을 직렬 처리
- 결과 0개이면 대기 후 한 번만 재시도
- 다시 0개이면 `BLOCKED`와 재개 위치 저장
- CAPTCHA·로그인·접근 통제를 자동 우회하지 않음
- 성공·차단·예외와 관계없이 `finally`에서 현재 실행이 띄운 임시 브라우저와 탭을 종료하고 정리 결과 기록

Browser Use 쿠팡 검색에서 발생한 `Access Denied`는 전체 IP 차단으로 단정하지 않는다. 현재 관찰상 Akamai의 세션·브라우저 판정과 관련된 접근 차이이며, 표시형 `nodriver` Chrome 조합에서 공개 검색 근거 수집이 동작했다.

- 공급처 이미지 사용권, KC·식품·화장품 등 규제 문서는 선택 후보에서 별도 확인해야 한다.
- 공통 프로젝트 폴더·상태 `1.0.0`은 구현됐지만 소싱→상세페이지 전체 필드 자동 승격은 다음 구현 단계다.
- React 빌드·단계 로직·로컬 API 테스트와 Browser Use 360px·800px 육안 QA는 통과했다. 실제 커머스 전문 스킬의 첫 버튼 실행은 사용자가 원하는 현재 단계에서 수행해야 한다.
- Codex 내장 콘솔 이벤트는 서버 메모리에만 보관되어 UI 서버 재시작 후 복구되지 않으며, 비대화형 실행에서 새 승인이 필요한 작업은 권한 확대 없이 실패할 수 있다.
- 5.3 하이브리드 계약은 회귀 테스트를 통과했지만 실제 SKU 최종 산출물과 쿠팡용 정적 렌더링은 아직 검증 전이다.
- ADR-014의 `phase_1`·`phase_2` 상태, 판매 신호 게이트와 2차 재승인 계약은 문서만 확정되었고 코드·테스트는 아직 없다.
