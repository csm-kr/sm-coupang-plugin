# 단계 레지스트리 지침

`stages.json`은 개발 단계와 필수 검증의 기계 판독 가능한 정본이다.

- 단계 ID는 소문자 kebab-case, 순서는 중복 없이 유지한다.
- 모든 단계는 구현 경로, 테스트 경로와 하나 이상의 실행 가능한 검증 명령을 가져야 한다.
- `test_paths`의 예시 파일은 반드시 해당 단계로 다시 라우팅되어야 한다.
- 계획 단계도 빈 명령을 두지 않는다. 첫 구현 시 테스트 디렉터리를 먼저 만든다.
- 단계 추가·이름 변경 시 루트 `AGENTS.md`, `docs/DEVELOPMENT-HARNESS.md`, ROADMAP을 함께 갱신한다.
